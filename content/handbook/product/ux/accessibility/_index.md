## 概要

アクセシビリティチームは、ユーザーにとって GitLab 製品と Pajamas Design System をよりアクセシブルにします。私たちは [Design Systems チーム](https://design.gitlab.com/) や、GitLab 全体のデザイナーおよびフロントエンドエンジニアと密接に協力しています。

## ミッション

私たちのミッションは、GitLab 製品がそのミッションを実現するために役割を果たし、私たちの世界を動かすソフトウェアに誰もが貢献し、共に作り上げられるようにすることです。

### 技術的な実装と Issue 解決

- 特定済みのアクセシビリティ Issue のバックログを順次対応する
- WCAG ガイドラインに従い、セマンティック HTML、ARIA、CSS、JavaScript を用いたアクセシビリティ対応を実装する
- アクセシビリティ準拠についてコードレビューを実施し、実用的なフィードバックを提供する
- 自動および手動テストを通じて新しい Issue を特定する

### プロセスとドキュメント

- ガイドラインやベストプラクティスを含むアクセシビリティに関するドキュメントを作成・維持する
- アクセシビリティテストおよび開発プロセスを改善する
- 体系的な Issue 解決のために、整理されたバックログを整備する
- アクセシビリティ標準に対する製品の評価により、Accessibility Conformance Report (ACR) を維持する

### トレーニングと支援

- UX および開発チームに対し、基本的なアクセシビリティのトレーニングとガイダンスを提供する
- デザイナー、エンジニア、PM が可能な限りアクセシブルな機能を提供できるよう支援する

### 部門横断的なコラボレーション

- 共有のアクセシビリティ目標に向けて、Design Systems チームと共に取り組む
- GitLab 全体のデザイナーおよびフロントエンドエンジニアと連携し、アクセシブルな機能を提供する
- 既知のアクセシビリティ Issue の優先順位付けと解決において、既存の製品チームを支援する

### カルチャー構築

- バグバッシュやその他の取り組みを通じて、実践的な学習機会を創出する
- オフィスアワー、Q&A セッション、特定のアクセシビリティ課題に関する深掘りセッションを開催する
- 障がいを持つユーザーと直接関わり、共感と理解を深める
- 組織全体でアドボカシーの文化を築く
