## リサーチおよび分析ツール {#research-and-analysis-tools}

### Dovetail {#dovetail}

リサーチの調査結果を管理・分析するためのリサーチリポジトリです。必要な場合は[アクセスをリクエスト](https://gitlab.com/gitlab-com/team-member-epics/access-requests/-/issues/new?issuable_template=Individual_Bulk_Access_Request)してください。

### Gong {#gong}

セールスの会話を記録し、分析を提供する会話インテリジェンスツールです。関心のあるトピックについて通話を検索したり、話をすべき顧客を特定したりできます。「Collaborator」ロールの[アクセスをリクエスト](https://gitlab.com/gitlab-com/team-member-epics/access-requests/-/issues/new?issuable_template=Individual_Bulk_Access_Request)してください。

### Highspot {#highspot}

セールスイネーブルメントや競合調査を含む Go-to-market 情報です。SSO 経由で利用できます。必要な場合は[アクセスをリクエスト](https://gitlab.com/gitlab-com/team-member-epics/access-requests/-/issues/new?issuable_template=Individual_Bulk_Access_Request)してください。詳細は [Highspot ハンドブックページ](/handbook/sales/field-communications/gitlab-highspot/)を参照してください。

### UX リサーチアーカイブ {#ux-research-archive}

[UX リサーチアーカイブ](https://gitlab.com/gitlab-org/uxr_insights)には、GitLab の UX リサーチャーが実施したすべてのリサーチが含まれています。

### Jobs to Be Done (JTBD) {#jobs-to-be-done-jtbd}

私たちは、顧客が達成しようとしているジョブの観点からプロダクトとソリューションを捉えるために [JTBD フレームワーク](/handbook/product/ux/jobs-to-be-done/)を使用しています。JTBD は、スコープの決定、方向性の検証、UX スコアカードによる既存エクスペリエンスの評価、ジャーニーマッピングなど、デザインプロセス全体を通じて使用されます。

### ツール {#tools}

#### Figma {#figma}

私たちはデザインとプロトタイピングに [Figma](https://www.figma.com/design/) を使用しています。[Pajamas UI kit](https://www.figma.com/file/qEddyqCrI7kPSBjGmwkZzQ/Pajamas-UI-Kit) には、GitLab のデザインシステム [Pajamas](https://design.gitlab.com/) のデザインアセット、コンポーネント、スタイルが含まれています。さらに、デザイナーがデザインファイルを GitLab の Issue に直接アップロードできる [Figma プラグイン](https://www.figma.com/community/plugin/860845891704482356/GitLab)も利用できます。すべてのプロダクトデザイナーは、オンボーディング中に Figma へのアクセス権を受け取るはずです。必要なアクセス権がない場合は、マネージャーに連絡してください。プロダクトデザイナーではないが View アクセス（コメントを残す機能を含む）を希望する場合は、無料の Figma アカウントを作成し、ステージグループのデザイナーに関連ファイルへのリンクを依頼してください。

Figma 管理者とそのロールスコープは次のように定義されています。

|管理者|スコープ|
|-----|-----|
|@tauriedavis|Tech stack オーナー、ユーザー管理（プロビジョニング／デプロビジョニング）、請求|
|@vkarnes|Tech stack オーナーのフォールバック|
|@jeldergl|ユーザー管理（プロビジョニング／デプロビジョニング）、design ops|
|@danmh|design ops|

#### Figjam {#figjam}

私たちは、デザインフィードバックの収集、ワークフローのマッピング、ブレインストーミング、アフィニティマッピングなど、視覚的なホワイトボードのようなワークスペースが必要なあらゆる場面で [Figjam](https://www.figma.com/figjam/) を使用しています。

UX 部門の全員とすべてのプロダクトマネージャーは、新しい Figjam ボードを作成できる Figma アカウントを取得できます。Figma アカウントを持っていないチームメンバーからフィードバックを得るために Figjam ボードを共有したい場合は、Share ダイアログから匿名リンクを送信できます。

#### AI ツール {#ai-tools}

AI をいつ使うべきか、ベストプラクティス、避けるべきこと、AI をヘルパーとして活用しながらユーザーを中心に据え続ける方法については、[UX における AI 利用](/handbook/product/ux/how-we-work/ai-usage/)を参照してください。

#### Dovetail {#dovetail-1}

私たちは、リサーチの調査結果を管理・分析するために [Dovetail](https://dovetailapp.com/) を使用しています。アクセスが必要な場合は、[アクセスリクエスト](https://gitlab.com/gitlab-com/team-member-epics/access-requests/-/issues)を提出してください。

#### Gong {#gong-1}

セールス向けの会話を記録し、それらの会話の分析やインサイトを提供する会話インテリジェンスツールです。UX チームメンバーが話をすべき顧客を特定したり、関心のあるトピックについて通話を検索したりするのに役立ちます。リクエストに応じて UX チームメンバーが利用できます。利用したい場合は[アクセスリクエスト](https://gitlab.com/gitlab-com/team-member-epics/access-requests/-/issues)を作成してください。「Collaborator」ロールをリクエストできます。

#### Highspot {#highspot-1}

セールスイネーブルメントや競合調査を含む Go-to-market に関する情報が含まれています。Highspot は SSO 経由でアクセスでき、リクエストに応じて利用できます。利用したい場合は[アクセスリクエスト](https://gitlab.com/gitlab-com/team-member-epics/access-requests/-/issues)を作成してください。詳細は [Highspot ハンドブックページ](/handbook/sales/field-communications/gitlab-highspot/)を参照してください。
