このガイドでは、一般的な Zendesk チケット指標を定義します。これらの指標はチケットライフサイクルのタイミングを追跡し、SLA ポリシー、レポート、パフォーマンス測定で使用されます。

**注記:** これらは Zendesk の公式定義です。GitLab では、内部レポート用にわずかに異なる計算や定義を使用する場合があります。SLA ポリシーの詳細については、[ドキュメント](/handbook/eta/css/zendesk/sla-policies/)を参照してください。

## 最初の返信時間 (FRT)

これは、チケットが作成されてからエージェントによる最初の公開返信を受け取るまでに要した時間です。

## 次の返信時間 (NRT)

これは、エンドユーザーからの公開返信を受けてから、エージェントによる公開返信を受け取るまでにチケットが要した時間です。

## 依頼者待機時間

**目的:** 顧客がエージェントのアクションを待つ時間を測定します（外部ソースを待つ時間を含む）。

これは、チケットが次のステータスにあった合計時間です。

- New
- Open
- On-hold

## エージェント作業時間

**目的:** チケットが直接的なエージェントのアクションを待つ時間を測定します。

これは、チケットが次のステータスにあった合計時間です。

- New
- Open

## エージェント待機時間

**目的:** チケットが顧客からの返信を待つ時間を測定します。

これは、チケットが次のステータスにあった合計時間です。

- Pending

## 保留時間

**目的:** チケットが外部ソース（つまりエージェントまたは顧客以外）を待つ時間を測定します。

これは、チケットが次のステータスにあった合計時間です。

- On-hold

## 合計解決時間

**目的:** チケットが作成されてから解決されるまでの時間を測定します。

これは、チケットが次のステータスにあった合計時間です。

- New
- Open
- Pending
- On-hold

## 定期更新

**目的:** チケットがエージェントからの公開返信の間に要する時間を測定します。

## 一時停止可能な更新

**目的:** チケットがエージェントからの公開返信の間に要した時間を測定します。

これは、チケットが次のステータスにある間に、エージェントからの公開返信の間に要する時間です。

- New
- Open
- On-hold

## 詳細な例

これらの指標がどのように連携するかを示すため、次のチケットライフサイクルを考えます。

**チケット #12345 のタイムライン:**

| 時刻 | イベント | ステータスの変更 |
|------|-------|---------------|
| 月曜 9:00 AM | 顧客がチケットを作成 | → New |
| 月曜 2:00 PM | エージェントが公開返信 | → Open |
| 月曜 3:30 PM | 顧客が返信 | → Open |
| 月曜 4:00 PM | エージェントが公開返信 | → Pending |
| 火曜 10:00 AM | 顧客が返信 | → Open |
| 火曜 11:00 AM | エージェントが Engineering からの情報を必要とする | → On-hold |
| 水曜 2:00 PM | Engineering が回答を提供 | → Open |
| 水曜 2:30 PM | エージェントが公開返信 | → Solved |

**計算された指標:**

| 指標 | 計算 | 値 |
|--------|-------------|-------|
| **FRT** | 月曜 9:00 AM → 月曜 2:00 PM（最初のエージェント返信） | 5 時間 |
| **NRT** | 月曜 3:30 PM → 月曜 4:00 PM（顧客の返信後のエージェント返信） | 30 分 |
| **依頼者待機時間** | New + Open + On-hold の時間<br>• New: 5 時間<br>• Open: 合計 8.5 時間<br>• On-hold: 27.5 時間 | **41 時間** |
| **エージェント作業時間** | New + Open の時間<br>• New: 5 時間<br>• Open: 8.5 時間 | **13.5 時間** |
| **エージェント待機時間** | Pending の時間<br>• 月曜 4:00 PM → 火曜 10:00 AM | **18 時間** |
| **保留時間** | On-hold の時間<br>• 火曜 11:00 AM → 水曜 2:00 PM | **27.5 時間** |
| **合計解決時間** | 月曜 9:00 AM → 水曜 2:30 PM<br>(New + Open + Pending + On-hold) | **53.5 時間** |
| **定期更新** | エージェントによるすべての公開返信の間の時間<br>• 月曜 2:00 PM → 月曜 4:00 PM = 2 時間<br>• 月曜 4:00 PM → 水曜 2:30 PM = 46.5 時間 | 最新: **46.5 時間** |
| **一時停止可能な更新** | New/Open/On-hold 中のエージェント返信の間の時間<br>• 月曜 2:00 PM → 月曜 4:00 PM（Open 中） = 2 時間<br>• 火曜 10:00 AM → 水曜 2:30 PM（Open + On-hold 中） = 28.5 時間 | 最新: **28.5 時間** |

**主な違いの説明:**

- **定期更新**は、エージェントによる返信の間のすべての時間をカウントします。これには Pending ステータス（顧客を待っているとき）が含まれます
- **一時停止可能な更新**は、Pending ステータスを除外し、New/Open/On-hold 中の時間（チケットがエージェント/会社側にあるとき）のみをカウントします
- **合計解決時間**（53.5 時間）= エージェント作業時間（13.5 時間）+ エージェント待機時間（18 時間）+ 保留時間（27.5 時間）- 一部の重複を調整
- **依頼者待機時間**（41 時間）= 顧客が私たちを待っているすべての時間（New + Open + On-hold）
