このガイドでは、Zendesk 内で項目（チケット、ユーザー、組織など）を検索するためのリファレンスを提供します。

## 検索の制限事項

検索では 1000 件を超える結果があると表示される場合がありますが、アクセスできるのは最初の 1000 件のみです（UI で使用しているソート方法に基づきます）。

## 基本検索

基本検索を行うには、Zendesk 内の任意のページの右上にある虫眼鏡アイコンをクリックします。検索内容を入力できるテキストボックスが表示されます。[高度な検索](#advanced-searching)で詳しく説明されている方法を使用しない場合、Zendesk 内のさまざまな項目に対して、入力したテキストを緩やかにテキスト一致させます（ファジーな大文字小文字を区別しない一致とも呼ばれます）。

検索クエリを入力したら、キーボードの Enter/Return キーを押します。

検索結果ページが表示されます。検索結果はタブ付きのセクションに分かれています:

- チケット
- 記事
- ユーザー
- 組織

これらの検索結果を共有したい場合は、白い `Copy link` ボタンを使用して、共有用に検索を再現するリンクを作成できます。

## 高度な検索

Zendesk の検索言語を使用して、より具体的な情報を検索する方法について説明します。すべての結果は、[基本検索](#basic-search)とほぼ同じように表示されます。基本検索で行う緩やかなテキスト一致よりも検索を絞り込む必要がある場合に最適です。

### 否定の使用

検索項目の先頭にマイナス記号を付けることで、否定検索できます。これにより、その項目に対して「NOT」スタイルの検索が実行されます。

たとえば、タグ `spam` を含まない項目を探す場合は、`-tags:spam` を使用します。

### タグで検索する

特定のタグを持つオブジェクトを検索するには、次の構文を使用します:

`tags:tag_name`

たとえば、タグ `skip_2fa_automation` を使用するオブジェクトを見つけるには、次を使用します:

`tags:skip_2fa_automation`

先頭にマイナスを付けることで、その否定を検索できます:

`-tags:tag_name`

複数のタグを持つオブジェクトを検索する場合は、二重引用符で囲みます:

`tags:"tag_name tag_name"`

たとえば、タグ `gold` と `sev1` の両方を含むすべてのオブジェクトを見つけるには、次を使用します:

`tags:"gold sev1"`

### 比較演算子

比較演算子は次のとおりです:

| 演算子 | 意味                  |
|:--------:|--------------------------|
| `:`      | 等しい                 |
| `<`      | より小さい                |
| `<=`     | 以下    |
| `>`      | より大きい             |
| `>=`     | 以上 |

### 組織で検索する

次の形式で組織を検索できます:

`organization:name_of_organization`

表示される内容はオブジェクトタイプによって異なります:

- チケットの場合、組織が所有するチケットを表示します
- ユーザーの場合、組織に関連付けられたユーザーを表示します
- 組織の場合、その組織を表示します

### チケットのみの方法

### ID で検索する

ID で検索するには、検索ボックスに ID 値だけを入力して検索します。これにより、ブラウザでチケットが開きます。

#### 優先度で検索する

チケットの優先度で検索するには、次の構文を使用します:

`priority{operator}name`

優先度の名前（および順序）は次のとおりです:

1. low
1. normal
1. high
1. urgent

これらの値は、上記の順序に基づいて順序比較されます

#### ステータスで検索する

チケットのステータスで検索するには、次の構文を使用します:

`status{operator}name`

ステータスの名前（および順序）は次のとおりです:

1. new
1. open
1. pending
1. hold
1. solved
1. closed

これらの値は、上記の順序に基づいて順序比較されます

#### 日付で検索する

日付を使用して検索するには、検索したい特定の日付オブジェクトを使用します。チケットの場合:

| オブジェクト | 関連する内容 |
|--------|-----------------|
| `created` | チケットが作成された日付 |
| `updated` | チケットの直近の更新日 |
| `due_date` | チケットの期日フィールドの値 |
| `solved` | チケットが解決済みになった日付（複数回解決済みに移動された場合は最新のもの） |

[比較演算子](#comparison-operators)と組み合わせて、`YYYY-MM-DD` の日付形式を使用して検索します。

たとえば、2023 年 12 月 18 日より後に作成されたすべてのチケットを見つけるには、`created>2023-12-18` を使用します。

#### カスタムチケットフィールドで検索する

カスタムチケットフィールドで検索するには、次の形式を使用します:

`custom_field_{id}:value`

ここで、`{id}` はチケットフィールド ID です。チケットフィールド ID 値を知っている必要があるため、これは直感的でない場合があります。チケットフィールドの ID 値を見つける際のサポートについては、Customer Support Systems チームに（Slack 経由で）相談するのが最適です。

#### 満足度評価で検索する

満足度評価に基づいてチケットを見つけるには、次の形式を使用します:

`satisfaction:value`

使用できる値は次のとおりです:

- bad - コメントなしの低評価
- badwithcomment - コメント付きの低評価
- good - コメントなしの高評価
- goodwithcomment - コメント付きの高評価
- offered - アンケートは送信済みですが、回答されていません

#### 依頼者で検索する

チケットの依頼者で検索するには、次の形式を使用します:

`submitter:search`

`search` に使用できるのは次のとおりです:

- ユーザーのメールアドレス
- ユーザーの名前
- ユーザーの ID
- `me` という語（自分自身を検索します）

#### リクエスターで検索する

チケットのリクエスターで検索するには、次の形式を使用します:

`requester:search`

`search` に使用できるのは次のとおりです:

- ユーザーのメールアドレス
- ユーザーの名前
- ユーザーの ID
- `me` という語（自分自身を検索します）

#### 担当者で検索する

チケットの担当者で検索するには、次の形式を使用します:

`assignee:search`

`search` に使用できるのは次のとおりです:

- エージェントのメールアドレス
- エージェントの名前
- エージェントの ID
- `me` という語（自分自身を検索します）

### 組織のみの方法

#### カスタム組織フィールドで検索する

カスタム組織フィールドで検索するには、次の形式を使用します:

`field_api_name{operator}query`

一般的な組織フィールド API 名は次のとおりです:

| フィールド                       | 用途                                   | Zendesk インスタンス |
|-----------------------------|--------------------------------------------------|------------------|
| `account_owner` | 組織の Account Manager | 両方 |
| `aar` | 組織の年間経常収益（ARR） | Global |
| `arr` | 組織の年間経常収益（ARR） | US Government |
| `assigned_se` | 組織の Assigned Support Engineer のユーザー ID | Global |
| `technical_account_manager` | 組織の Customer Success Manager | 両方 |
| `support_level` | 組織の最高のサポートレベル | 両方 |
| `health_score` | 組織の Gainsight ヘルススコア | 両方 |
| `seats_decimal` | 組織の最大ライセンスシート数 | Global |
| `number_of_seats` | 組織の最大ライセンスシート数 | US Government |
| `salesforce_id` | 組織の 18 文字の Salesforce アカウント ID | 両方 |
| `sfdc_short_id` | 組織の 15 文字の Salesforce アカウント ID | 両方 |
| `solutions_architect` | 組織の Solutions Architect | US Government |

例:

年間経常収益が 1000 未満の組織を検索する場合:

`aar<1000`

### ユーザーのみの方法

#### メールアドレスで検索する

メールアドレスでユーザーを見つけるには、次の形式を使用します:

`email:email_address`

たとえば、Zendesk でメールアドレスが `jcolyer@example.com` のユーザーを見つけるには、`email:jcolyer@example.com` を使用します。

#### カスタムユーザーフィールドで検索する

カスタムユーザーフィールドで検索するには、次の形式を使用します:

`field_api_name:query`

一般的なユーザーフィールド API 名は次のとおりです:

- `gitlab_username` - エージェントの GitLab.com ユーザー名
- `gitlab_user_id` - エージェントの GitLab.com ユーザー ID

たとえば、GitLab.com ユーザー名が jcolyer であるユーザーを見つけるには、次を使用します:

`gitlab_username:jcolyer`

### 高度な検索項目を組み合わせる

複数の高度な検索項目をスペースで区切ることで、より絞り込んだ検索を作成できます。

たとえば、タグ `customer` を含み、ステータスが `new` のすべてのチケットを見つけるには、`tags:customer status:new` を使用します。

### 例

#### 例 1

タスク:

- 推奨されるサポート地域は AMER（ID 値 `360018253094` を使用するカスタムフィールド）
- 組織のサポートレベルは ultimate
- 満足度評価は低評価（コメントあり）

`custom_field_360018253094:americas__usa tags:ultimate satisfaction:badwithcomment`

#### 例 2

タスク:

- new、solved、closed を除くすべてのチケットステータス
- 担当者は Jason
- SaaS Account の問題タイプは Namesquatting（ID 値 `360011793260` を使用するカスタムフィールド）

`-status:new -status:solved -status:closed assignee:Jason custom_field_360011793260:namesquatting_requests`

#### 例 3

タスク:

- Salesforce ID に基づいて Zendesk 内の組織を見つける

Salesforce アカウントには実際には 2 つの ID 形式があります。標準の 18 文字の値と短縮された 15 文字の値です。Zendesk 組織には両方の値が含まれているため、どちらを使用してもアカウントを見つけられます:

- 18 文字の ID 値に基づく場合:
  `salesforce_id:ABCDEFGH0123456789`
- 15 文字の ID 値に基づく場合:
  `sfdc_short_id:ABCDEFGH0123456`

アスタリスクを使用してワイルドカード検索もできます。15 文字の値がある場合は、次のようにも検索できます:

`salesforce_id:ABCDEFGH0123456*`

#### 例 4

タスク:

- アカウントの Salesforce ID に基づいて L&R IR チケットを見つける

[例 3](#example-3)で説明したとおり、特定の Salesforce アカウントに対して取得できる値は 2 つあります。L&R IR には、申請対象の Salesforce アカウントを見つけるために使用できるカスタムフィールドがありますが、2 つのどちらが使用されたかを知るのは難しい場合があります。そのため、ワイルドカード検索では 15 文字の値を使用するのが最適です。18 文字の値しかない場合は、ID の最後の 3 文字を削除します（たとえば、`ABCDEFGH0123456789` の場合は、最後の 3 文字を削除して `ABCDEFGH0123456` にします）。その場合、次のいずれかの検索を実行します（検索する Zendesk インスタンスによって異なります）:

- Zendesk Global:
  `custom_field_6978327875612:ABCDEFGH0123456*`
- Zendesk US Government:
  `custom_field_11717220820372:ABCDEFGH0123456*`
