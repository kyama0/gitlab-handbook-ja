## 監査を理解する

### 監査とは

監査とは、さまざまなプラットフォームへのアクセス権を誰がどのように持っているかをレビューするプロセス（および中核的な責任）を指します。

### 監査を実施する理由

監査を実施する理由は多数ありますが、私たちにとって最も大きな理由は次のとおりです:

- エージェント／ユーザー領域でセキュリティ問題が発生していないことを確認する。
- 調達と更新のために正確な情報を確保する。
- 管理するさまざまなシステムを誰が使用しているかのレビューについて、ベストプラクティスに従うことを確認する。

### 監査対象

- Customer Support Systems プロジェクト
- Zendesk Global
- Zendesk US Government

### 監査を実施する時期

各四半期の初日に、[System Audits](https://gitlab.com/gitlab-support-readiness/system-audits)プロジェクトを通じて Issue が生成されます。Issue の作成後、監査を実施します。

## Customer Support Systems プロジェクト

これは `bin/ops_project_audit` スクリプトで行います。スクリプトを実行すると、次の処理が行われます:

- グループ内のすべてのプロジェクトのリストを取得する
- 見つかったプロジェクト内のすべての保護ブランチを取得する
- 見つかったプロジェクト内のすべてのマージリクエスト設定を取得する
- 次を含む問題について、すべてのプロジェクトを分析する:
  - 不正な作成者承認設定
  - 不正なコミット承認設定
  - 不正な承認のオーバーライド設定
  - 不正な承認保持設定
  - マージできる人のリストが不正
  - デフォルトブランチにプッシュできる人のリストが不正
- すべての検出結果を詳述する Issue を作成する

生成された Issue の対応担当者は、その Issue に含まれる問題をレビューして調査します。

問題を是正した後、Issue をクローズします。

## Zendesk Global の監査

これは `bin/zendesk_global` スクリプトで行います。スクリプトを実行すると、次の処理が行われます:

- Zendesk のすべてのエージェント（つまり、ロールが `admin` または `agent` のユーザー）のリストを取得する
- 3 つのユーザーリストを作成する:
  - ライトエージェント（カスタムロール ID が `360004984553` のユーザー）
  - 管理者（カスタムロール ID が `360004957599` のユーザー）
  - フルエージェント（ライトエージェントおよび管理者のリストにないユーザー）
- Zendesk のすべてのエージェントをループ処理し、停止状態のエージェントがいないかを確認する（停止状態であってはなりません）
- 次の内容を詳述する Issue を作成する
  - すべてのライトエージェントのリスト
  - すべてのフルエージェントのリスト
  - すべての管理者のリスト
  - すべての停止状態のエージェントのリスト

生成された Issue の対応担当者は、その Issue に含まれる問題をレビューして調査します。

問題を是正した後、Issue をクローズします。
