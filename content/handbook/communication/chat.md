## はじめに

GitLab では、Slack はお互いのコミュニケーションに欠かせないツールです。リアルタイムのコミュニケーションを可能にしながらも、非同期のマインドセットを維持することも大切にしており、GitLab チームメンバーは「通知なし」に設定し、常に他者からリアルタイムで返信が来ることを期待しないようにしています。

GitLab のさまざまな領域に対応するチャンネルグループがあります。このページでは、それらのチャンネルグループのいくつかを説明します。

## 検索

`Search GitLab` ツールバーを使用するか、`#channel-name` 形式のチャンネル名を使用するか、**または**左サイドバーで `More` をクリックして `Channel browser` をクリックしてチャンネルを検索できます。

## チャンネル

### 主要な Slack チャンネル

以下のアルファベット順のリストは、GitLab の多数の Slack チャンネルの中からいくつかをピックアップし、チームメンバーが特定の質問をしたり、さまざまなトピックについて議論するための最適な場所についてガイダンスを提供するものです。すべての利用可能なチャンネルの閲覧方法については、Slack の[ヘルプセンター](https://slack.com/help/articles/205239967-Join-a-channel)を参照してください。

| チャンネル | 目的 |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `#company-fyi` | 公式な会社発表。投稿には許可レベルの制限があり、シグナルが高い。すべての GitLab チームメンバーはこのチャンネルに自動追加される。 |
| `#whats-happening-at-gitlab` | リマインダー、イベント、プロジェクト更新など、GitLab の**ほとんどまたはすべて**のメンバーにとって重要かつ有用な非公式な更新のために、すべてのチームメンバーに公開。すべての GitLab チームメンバーはこのチャンネルに自動追加される。 |
| `#diversity_inclusion_and_belonging` | GitLab の最新の[多様性・包括性・帰属意識](/handbook/company/culture/inclusion/)イニシアチブの最新情報を得て、環境をさらにインクルーシブにする方法についてフィードバックや考えを共有する。 |
| `#expense-reporting-inquiries` | [経費](/handbook/finance/spending-company-money/)（例：Navan）に関する質問用。 |
| `#git-help` | ターミナルでの Git の使用に関する具体的な質問。 |
| `#is-this-known` | 既存の問題に対する既存の Issue を見つけるためのヘルプ。 |
| `#it_help` | IT 全般の質問やセットアップのトラブル（2FA、アカウントなど）がある場合はこのチャンネルでヘルプリクエストを作成する。 |
| `#mr-buddies` | マージリクエストに関する質問用。 |
| `#new_team_members` | 新しい GitLab チームメンバーが会社に自己紹介したり、既存のチームメンバーが新入社員と情報を共有したりする場所。 |
| `#new-vacancies` | 内部の機会およびすべての求人情報をすべての GitLab チームメンバーが認識するための場所。[紹介プロセス](/handbook/hiring/referral-process/)ページで紹介方法を確認してください。 |
| `#payroll` | [給与](/handbook/finance/payroll/)と[請負業者の請求書](https://internal.gitlab.com/handbook/finance/payroll/#non-us)（ADP など）に関する質問用。 |
| `#safe` | SAFE |
| `#security` | セキュリティ |
| `#talent-acquisition` | [紹介](/handbook/hiring/referral-process/)、[採用プロセス](/handbook/hiring/)、候補者の状況に関する質問用。 |
| `#team-member-updates` | 異動/昇進、新しい GitLab チームメンバーの入社、勤続記念日などの最新情報。 |

### 一般チャンネル {#random-room}

多くの一般チャンネルがあります。例として：

- **[`#questions`](https://gitlab.slack.com/archives/questions)**: GitLab 全体に質問を投稿してみましょう…そして答えが見つかったら文書化しましょう :)
- **[`#thanks`](https://gitlab.slack.com/archives/thanks)**: GitLab チームメンバーが、素晴らしい仕事をした、顧客を助けた、または助けてもらった GitLab チームメンバーに[感謝を伝える](/handbook/communication/#say-thanks)場所！マネージャーがチームへの裁量的ボーナスを認める場所でもあります。
- **[`#office-today`](https://gitlab.slack.com/archives/office-today)**: GitLab は[オールリモート](/handbook/company/culture/all-remote/)組織です。あなたの今日のオフィスはどこですか？
- **[`#remote`](https://gitlab.slack.com/archives/travel)**: リモートワークに関するニュース、考え、フィードバック、その他何でも共有！[GitLab のリモートワークへのアプローチ](/handbook/company/culture/all-remote/)をさらに学んでください。
- **[`#travel`](https://gitlab.slack.com/archives/travel)**: すべての[旅行](/handbook/company/culture/all-remote/)について話し合う場所。
- **[`#women`](https://gitlab.slack.com/archives/travel)**: メンバーとアライのための従業員リソースグループ。
- **[`#values`](https://gitlab.slack.com/archives/travel)**: GitLab バリューのイテレーションについて議論するチャンネル。
- **[`#random`](https://gitlab.slack.com/archives/random)**: 仲間の GitLab チームメンバーと交流し、ランダムな話題を共有しましょう。

### チームチャンネル

特定のチームへ Issue を提起するには、以下のよく使用されるチャンネルを使用してください：

- **`#it_help`**: IT 関連の問い合わせ、Okta、ハードウェア・ソフトウェアの問題、システムアクセスリクエスト、ユーザー権限など
- **`#talent-acquisition`**: 紹介に関する質問、候補者の調整、一般的な人材獲得など

### チャンネルカテゴリ

#### アカウントチャンネル（a_）

これらのチャンネル（`a_` プレフィックス付き）は、顧客組織のアカウントに関してチームメンバーが協働・コミュニケーションするためのものです。顧客によっては内部向け、プライベート、または顧客自身の Slack ワークスペースに接続して顧客と共有するものもあります。共有チャンネルと内部チャンネルを明確に区別するために、`a_XYZ-internal` と `a_XYZ-external` の命名規則を使用することを推奨します。

新しいチャンネルのリクエストや既存チャンネルの変更には、[Slack リクエスト](https://gitlab.com/gitlab-com/team-member-epics/access-requests/issues/new?issuable_template=Slack_Request)フォームを使用してください。

#### 外部チャンネル（ext-）

これらのチャンネル（`ext-` プレフィックス付き）は、ベンダーやパートナーとのコミュニケーションに使用されます。通常、GitLab の Slack ワークスペースと外部組織の Slack ワークスペース間の共有チャンネルです。

**例**

- **`#ext-gitlab-abc`**: プロジェクト協働とサポートのための顧客/ベンダー ABC との共有チャンネル。

外部チャンネルを使用する際は、GitLab の外部コミュニケーションガイドライン（情報を [SAFE](/handbook/legal/safe-framework/) に保つことを含む）に従ってください。1 年間アクティビティがない外部チャンネルは CorpSec によって削除されます。

#### フィーチャーチャンネル（f_）

これらのチャンネルは、GitLab に存在する、または現在構築中の特定の機能に興味があるチームと GitLab チームメンバー向けです。

**例**

- **[`#f_web_ide`](https://gitlab.slack.com/archives/f_web_ide)**: [Web IDE](https://about.gitlab.com/blog/2018/06/15/introducing-gitlab-s-integrated-development-environment/) についてのアイデアとフィードバックを共有する。
- **[`#f_ldap`](https://gitlab.slack.com/archives/f_ldap)**: [LDAP プロバイダーとのインテグレーション](https://docs.gitlab.com/ee/administration/auth/ldap/index.html)について話し合う。

#### グループチャンネル（g_）

グループチャンネル（`g_` プレフィックス付き）は[DevOps ステージグループ](/handbook/product/categories/#hierarchy)およびその他の[エンジニアリング部門とチーム](/handbook/engineering/#engineering-departments)に対応しています。

**例**

- **[`#g_create`](https://gitlab.slack.com/archives/g_create)**: [Create](/handbook/product/categories/#create-stage) 開発グループ用チャンネル。
- **[`#g_create_source-code`](https://gitlab.slack.com/archives/g_create_source-code)**: [ソースコード](/handbook/product/categories/#source-code-group)開発グループ用チャンネル。
- **[`#g_geo`](https://gitlab.slack.com/archives/g_geo)**: [Geo グループ](/handbook/engineering/infrastructure-platforms/tenant-scale/geo/)専用。

#### 場所チャンネル（loc_）

これらのチャンネルは、世界の同じ地域にいる GitLab チームメンバーが集まりやその他の場所固有の事項について話し合うのに役立ちます。世界の別の場所を訪問する際にも役立ちます。近くに GitLab チームメンバーがいるかもしれません！旅行中にそこに住む GitLab チームメンバーとつながるために一時的にチャンネルに参加してください。

新しいチャンネルがある場合は、MR でアルファベット順に追加してください。

（場所チャンネルの詳細一覧は原文をご覧ください）

#### プロジェクトチャンネル（proj_）

これらのチャンネルは一時的なもので、特定のプロジェクトに関するコミュニケーションに焦点を当てるために使用されます。プロジェクトが完了するとアーカイブされ、`proj_` で検索することで Slack で見つけることができます。

#### ワーキンググループチャンネル（wg_）

これらのチャンネルは一時的なもので、特定のワーキンググループに関するコミュニケーションに焦点を当てるために使用されます。ワーキンググループの終了条件が満たされるとアーカイブされ、`wg_` で検索することで Slack で見つけることができます。

#### ステージチャンネル（s_）

ステージチャンネル（`s_` プレフィックス付き）はサブ部門内のステージに対応しています。

**例**

- **[`#s_enablement`](https://gitlab.slack.com/archives/s_enablement)**: [Enablement ステージ](/handbook/product/categories/#enablement-stage)用。
- **[`#s_create`](https://gitlab.slack.com/archives/s_create)**: [Create ステージ](/handbook/product/categories/#create-stage)用。

#### ソーシャルグループ

週次の会社グループ通話に加えて、GitLab にはソーシャル Slack グループがあります。ソーシャルグループは、共通の興味、趣味、ライフスタイルのもとにチームメンバーをつなぐ Slack チャンネルです。テニスクラブ、ゲーム、貝殻収集、映画好き、信仰グループ、フットボールなど。共感するものには自由に参加してください。

**例：**

- **[`#all-caps`](https://gitlab.slack.com/archives/C01BC085AVB)**: すべて大文字での気軽な会話。
- **[`#bookclub`](https://gitlab.slack.com/archives/C0GDX6MK6)**: 読書好きがつながり、おすすめ本を共有し、最新の読書について話し合う場所。
- **[`#cats`](https://gitlab.slack.com/archives/cats)、[`#dog`](https://gitlab.slack.com/archives/dog)、[`#cute-animal-photos`](https://gitlab.slack.com/archives/cute-animal-photos)**: ペットのかわいい写真を見せてください！
- **[`#cooking`](https://gitlab.slack.com/archives/cooking)**: お気に入りのレシピを共有し、手作り料理を自慢しましょう。
- **[`#gaming`](https://gitlab.slack.com/archives/gaming)**: ゲーム（ボードゲームとビデオゲーム）について話し合う。
- **[`#lgbtq`](https://gitlab.slack.com/archives/lgbtq)**: GitLab の LGBTQ の方々とアライが話し合い、サポートし合うスペース。
- **[`#mental_health_aware`](https://gitlab.slack.com/archives/mental_health_aware)**: メンタルヘルスは[非常に重要](/handbook/company/culture/all-remote/mental-health/)です。懸念事項を話し合ったり、一人で抱え込んでいる場合に話し相手を見つける場所。
- **[`#random`](https://gitlab.slack.com/archives/random)**: 仲間の GitLab チームメンバーと交流し、ランダムな話題を共有しましょう。
