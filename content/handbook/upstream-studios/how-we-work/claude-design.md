このページでは、GitLab の UX 業務における **Claude Design を使うためのベストプラクティスとガイドライン**を扱います。
より広範な [AI usage](/handbook/upstream-studios/how-we-work/ai-usage/) のガイダンスを補完し、Claude のデザイン志向の機能に特化したアドバイスを提供します。

Claude Design とは、デザイン関連のタスクに Claude（Anthropic の AI アシスタント）を使うことを指します。UI コンセプトの生成、デザインのレビュー、UI テキストの作成、プロトタイプの作成、エンドツーエンドのデザインプロセスのサポートなどが含まれます。

## はじめに

Claude Design を使う前に、必ず次を読んでください。

- [AI usage](/handbook/upstream-studios/how-we-work/ai-usage/) — GitLab の UX 業務における一般的な AI ガイダンス
- [AI usage requirements and FAQs](https://internal.gitlab.com/handbook/company/ai-at-gitlab/#usage-requirements-and-faqs)
- [General purpose AI tool usage requirements](https://internal.gitlab.com/handbook/ai-security-at-gitlab/ai-tool-usage-requirements/)

Claude は私たちの[承認済み AI ツール](https://internal.gitlab.com/handbook/company/ai-at-gitlab/#approved-ai-tools)の 1 つです。そこで定義されているデータ分類レベルを常に尊重してください。

## Claude Design を使うべきとき

Claude Design は、デザインの判断やユーザーリサーチの代替ではなく、**創造性を加速させる存在であり、思考のパートナー**として最も効果を発揮します。

### アイデア出しと探索

- 方向性を固める前に、問題の領域を探索するために複数のデザインの方向性をすばやく生成します。
- 見落としていたかもしれないエッジケース、エラー状態、空の状態をブレインストーミングします。
- 議論の出発点として、ユーザーフローやインタラクションパターンの草案を作成します。
- 機能が異なるユーザーコンテキストやペルソナでどう見えるかを探索します。

### UI テキストとコンテンツ

- エラーメッセージ、ツールチップ、空の状態、マイクロコピーを作成し、繰り返し改善します。
- [GitLab のブランドボイス](https://design.gitlab.com/content/voice-tone)に合わせてトーンを調整します。コンテンツをより明確に、より短く、より人間味のあるものにするよう Claude に依頼します。
- 実験のために UI テキストの A/B テストのバリエーションを生成します。
- ドキュメントやハンドオフのメモのために、デザインの意図を平易な言葉に変換します。

## Claude Design を使うべきでないとき

- **ユーザーリサーチを決して代替しない**: Claude は実際のユーザーの行動、感情、コンテキストをシミュレートできません。デザインの決定は必ず実際のユーザーで検証してください。
- **AI の初期出力を最終デザインとして使わない**: Claude の出力は出発点として扱ってください。ユーザーに届くすべてのデザインは、人間のデザイナーがレビューし、洗練させ、責任を持つ必要があります。
- **慎重さが求められる、影響の大きい決定には使わない**: 脆弱なユーザーのセキュリティ、プライバシー、アクセシビリティに影響するデザインには、慎重な人間の監督が必要です。
- **Pajamas Design System を省略しない**: Claude は [Pajamas](https://design.gitlab.com/) に沿わないパターンを提案することがあります。出力が私たちのデザインシステムに準拠していることを必ず確認してください。

## ベストプラクティス

### 効果的なデザインプロンプトを書く

Claude の出力の品質は、プロンプトの品質に大きく依存します。豊富なコンテキストを提供してください。

| 含めるもの | 例 |
|---|---|
| **役割とオーディエンス** | "あなたは DevSecOps プラットフォームに取り組む UX デザイナーです。ユーザーは CI/CD パイプラインを管理するシニアソフトウェアエンジニアです。" |
| **具体的なタスク** | "マージリクエストのパイプライン中にセキュリティスキャンが失敗したときのエラーメッセージを 3 つのバージョンで書いてください。" |
| **デザインの制約** | "GitLab の Pajamas デザインシステムに従ってください。センテンスケースを使ってください。メッセージは 80 文字以内に収めてください。" |
| **例** | "私たちが使っている既存のエラーメッセージは次のとおりです: '…'。このトーンと形式に合わせてください。" |
| **成功基準** | "メッセージは、何が失敗したのか、なぜそれが重要なのか、ユーザーが次に何をできるのかを説明する必要があります。" |

ゼロからやり直すのではなく、会話を通じてプロンプトを繰り返し改善してください。出力を洗練させるためにフォローアップの質問をします。

### デザインの品質を保つ

- **出力に責任を持つ**: Claude が生成したデザインコンテンツを共有する前に、レビューし、編集し、責任を持ってください。
- **Pajamas と照合する**: 提案されたコンポーネント、パターン、言葉が [Pajamas Design System](https://design.gitlab.com/) に沿っていることを確認してください。
- **実際のユーザーでテストする**: AI が生成したデザインは仮説です。ユーザビリティテストやその他のリサーチ手法を通じて検証してください。
- **アクセシビリティの主張をファクトチェックする**: Claude はアクセシビリティの改善を提案することがありますが、必ず [WCAG ガイドライン](https://www.w3.org/WAI/standards-guidelines/wcag/)と照合し、支援技術でテストしてください。
- **テンプレート的な出力を避ける**: Claude の出力がありきたりに感じられる場合は、実際のプロダクトからより具体的なコンテキストや例を提供してください。

### コラボレーションと共有

- Claude を活用したデザインの決定を Issue や MR に記録し、レビュアーがプロセスを理解できるようにします。
- チームの集合知を高めるために、成功と失敗の両方をチームと共有します。

### 透明性を保つ

一般的な AI usage ガイドの[透明性に関するベストプラクティス](/handbook/upstream-studios/how-we-work/ai-usage/#transparency)に従ってください。

- 成果物のどの部分で Claude の助けを使ったかを記録します。
- リサーチレポート、デザイン仕様、ユーザーに向けた決定における AI の関与を開示します。
- AI の使用は、価値を加えるために言及するものであり、品質の低さの言い訳や責任回避のためではありません。

## アクセシビリティの考慮事項

Claude Design はアクセシビリティの問題を特定するのに役立ちますが、適切なアクセシビリティテストの代わりにはなりません。

- 一般的なアクセシビリティの問題（カラーコントラスト、フォーカス順序、ラベルの明確さ）についてデザインをレビューするよう Claude に依頼します。
- 提案は必ず [GitLab のアクセシビリティガイドライン](/handbook/product/ux/accessibility/)と [Pajamas のアクセシビリティドキュメント](https://design.gitlab.com/accessibility/overview)と照合してください。
- 実際の支援技術でテストし、可能であれば障害のあるユーザーでテストしてください。
- アクセシブルな UI テキストの代替案（alt テキスト、ARIA ラベル、説明的なボタンラベル）を出発点として Claude に生成させ、その後レビューしてください。

## 推奨事項

| ✅ する | ❌ しない |
|---|---|
| アイデア出しと探索を加速させるために Claude を使う | Claude の出力をレビューせずに最終デザインとして使う |
| プロンプトに豊富で具体的なコンテキストを提供する | あいまいなプロンプトを使い、最初の結果を受け入れる |
| 出力を Pajamas と GitLab のガイドラインと照合する | デザインシステムの確認を省略する |
| デザインを実際のユーザーで検証する | Claude の出力をユーザーによる検証として扱う |
| プロンプトと学びを記録して共有する | 成功したプロンプトを自分だけのものにする |
| 自分の作業における AI の関与を開示する | Claude を使ったことを隠す |
| 会話を通じて繰り返し改善する | 毎回ゼロからやり直す |
| Claude を生成だけでなく批評者として使う | Claude を作成にのみ使い、レビューには使わない |
