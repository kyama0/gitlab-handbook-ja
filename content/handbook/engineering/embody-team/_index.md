Embody チームは実験的な機能に特化した小規模なチームです。

## ビジョン

果敢な実験と迅速なイテレーションを通じて、次世代の DevSecOps イノベーションを発見・開発します。

## 優先事項

現在の優先事項は、ユーザーが GitLab で使いたいオブザーバビリティ機能を理解することです。

## スキル

カバーすべきドメインの範囲が広いため、さまざまな専門知識とスキルが必要です：

| テクノロジースキル | 関心領域    |
|-------------------|----------------------|
| Ruby on Rails     | バックエンド開発  |
| Go                | バックエンド開発  |
| Vue, Vuex, React  | フロントエンド開発 |
| GraphQL           | *各種*            |
| SQL (PostgreSQL)  | *各種*            |
| Docker/Kubernetes | *各種*     |

## 誰でも貢献できます

GitLab では[誰でも貢献できる](/handbook/company/mission/#contribute-to-gitlab-application) ことを目標としています。これは GitLab チームメンバーと、コミュニティコントリビューションを通じたより広いコミュニティに適用されます。私たちはあらゆる機能へのコントリビューションを歓迎しますが、初めてのコントリビューターはより小さな機能から始めることを好む場合があることを認識しています。これをサポートするために、初めてのコントリビューターや Software Supply Chain Security のドメインに新しいコントリビューターに適している可能性がある `quick wins` のリストを維持しています。

コントリビューターが EE ライセンスを必要とする場合は、コミュニティコントリビューターのワークフローページの [GitLab Enterprise Edition（EE）へのコントリビューション](/handbook/marketing/developer-relations/engineering/community-contributors-workflows/#contributing-to-the-gitlab-enterprise-edition-ee) セクションに案内できます。
