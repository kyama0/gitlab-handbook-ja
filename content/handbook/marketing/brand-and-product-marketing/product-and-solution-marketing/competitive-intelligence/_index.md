## GitLab における競合インテリジェンス

GitLab の競合インテリジェンス機能には 2 つの主要な構成員がいます: (1) 見込み客と顧客 (2) セールス活動を支援するパートナーを含むセールス組織。

**見込み客と顧客**: GitLab のコア価値である**[透明性](/handbook/values/#transparency)**に従い、競合インテリジェンスは、エンドツーエンドの DevOps ソリューションを探している人なら誰でも、顧客のビジネス目標に役立つ正しい偏見のない決定を下すために必要な情報を持っていることを保証します。このために、すべての製品比較ページはゲートされておらず、**[DevOps ツール比較ページ](https://about.gitlab.com/why-gitlab/)** で利用可能です。GitLab は、競合インテリジェンスの透明性が、十分に情報を得た顧客を作り、DevOps の旅を成功させるために必要な情報をチームに与えると信じています。

**セールスチーム**: もう 1 つの GitLab のコア価値は **[結果](/handbook/values/#results)** です。セールスは、GitLab が結果を達成するのに役立つ重要な構成員です。したがって、競合インテリジェンス資料の一部は、セールスチームが見込み客に効果的に販売するのを助けることに焦点を当てており、それによって顧客のビジネスニーズを満たす価値を提供しています。

## チームチャーター

競合インテリジェンスチームは、GitLab のアクションと方向性をガイドするのを助けるために、競合他社のアクションのタイムリーで洞察に満ちた分析を提供します。

- 私たちは、洞察をアクションに変えるフィードバックループを改善するのに役立つようにクロスファンクショナルに作業し、顧客のために製品を改善することを目的として主要部門により良い情報を提供するために競合データを使用します。
- 私たちは、競合他社のナラティブを解体するためのより良いツールでセールスチームをエンパワーする最前線の両方で、また、市場の動きを予測するのに役立つ戦略的方向性をガイドする最高レベルのシニアリティで作業しています。
- 私たちは、勝利と敗北の理由を測定し、主要な弱点を表面化、トリアージ、解決するのを助けながら、強みを祝うことを保証します
- 私たちは、選択する DevSecOps プラットフォームとして GitLab が市場およびソートリーダーであることを保証するために、戦略的情報を使用して全社のシニアリーダーシップに事前に情報を提供し、会社戦略をガイドします
- 私たちは、市場からのタイムリーな情報を取得し、それを使用して GitLab がどのように行動、反応し、競合他社を考慮するかについての評価をガイドします

## 比較アセット

### Crayon

#### Crayon について

すべての内部競合インテリジェンス（競合カード、内部ウェビナー、ドキュメント）は、競合カードをホストする[Crayon](/handbook/marketing/brand-and-product-marketing/product-and-solution-marketing/competitive-intelligence/#crayon) 競合インテリジェンスツールを活用して、[Highspot ページ](https://gitlab.highspot.com/spots/615dd78be6303bc4429268a1) でホストされています。

Crayon の詳細については、[内部ハンドブックの競合インテリジェンスページ](https://internal.gitlab.com/handbook/marketing/brand-and-product-marketing/) を参照してください

Crayon へのアクセスは、すべての当社のセールス、プロダクト、エンジニアリング、マーケティングチームに開かれています。Okta タイル経由で Crayon にアクセスできます（リストに表示されない場合は Okta 内で検索してください）。GitLab のコラボレーションの価値の精神で、すべてのアカウントには「コラボレーター」レベルのアクセスが与えられ、すべてのユーザーが競合インテリジェンスチームと同じ方法でコンテンツを作成および編集できることを意味します。必要に応じて、ユーザーが「ビューア」アクセスにダウングレードされる場合があります。

これらのチームに所属しておらず、アクセスが必要な場合は、Slack で @jkempton にリクエストを連絡してください。

GitLab のイテレーションとコラボレーションの価値の精神で、Crayon は GitLab のチームメイトが、競合インテリジェンスチームのみによってゲートキープされるのではなく、競合インテリジェンスを発見した場所と時に貢献および開発できるように設計されています。その点で、チームは、競合カードでの競合インテリジェンスニュースに更新、修正、修正、コメントを行い、競合ボードでコラボレーションして、誰もが最高の最新情報を持っていることを保証することが奨励されます。Crayon にはまた、競合インテリジェンスをソースし表面化するのを助けるために舞台裏で働く専任のアナリストもいます。Crayon ツール内で当社のアナリストからのコメントを見ることがあります。

Crayon プラットフォームの使用方法に関する質問については、ヘルプを求めて '@jkempton' に連絡する前に、Crayon の[独自の FAQ](https://support.crayon.co/hc/en-us) を使用してください。

ツールの「ヘルプ」機能は Crayon ツールの使用に関する質問のためにのみ使用すべきであり、GitLab の競合インテリジェンスチームに向けた一般的な質問のためではありません（例えば、「この競合相手についてヘルプしてもらえますか」のような質問は、Crayon を介して質問するのではなく、'#competition' Slack チャンネルで共有されるべきです）

#### Crayon の使用方法

GitLab のセールス、プロダクト、エンジニアリング、マーケティングチームメンバーは、以下を含めて Crayon に貢献できます:

- 新しい競合カードの追加
- 異議処理の質問と解決の追加
- 製品リリースまたは注目すべき競合変更の強調表示

Crayon に競合情報を追加、削除、または修正する場合は、以下のプロセスに従ってください:

1. コンテンツを追加するための適切な競合カードを見つけます。情報の重要性を評価することが重要です: 1 つの競合相手だけに影響を与えますか？それはより広いテーマの一部ですか？情報を他の競合カードに複製する必要がありますか？
2. 新しいコンテンツが作成されると、再公開されるまで情報は更新されません。GitLab 内の MR のプロセスと同様に、行われた追加/削除を確認するために、カードのコメントセクションで @Jkempton にタグを付けてください。
3. @jkempton は、コンテンツの変更をレビューし、適切な場合は編集し、カードを再公開します。

Crayon は、フィールドが素早く参照するための競合情報をホストするように設計されていることを覚えておいてください。これは広範な説明を持つ情報の辞書ではなく、真実で単純な説明と直接的にする必要があります。

### Clozd

Clozd は、当社のサードパーティの勝敗インタビューパートナーです。Clozd は、GitLab を評価した顧客や見込み客との深いダイブインタビューを行うことを可能にします。Clozd のサービスを利用して、当社のマーケティング、セールスプロセス、製品機能の偏見のないビューを得ること、および以下の課題を解決することを保証しています:

- 一部の内部データは、なぜ取引に勝ち、敗北するかについて十分な正確さを与えない場合があります。または少なくとも高レベルの概要を提供します
- 異なるトピックに関する数百の Issue がある場合、どの製品機能が優先順位を取るべきかを知ることは難しい場合があります
- 競合状況で誰が最大の脅威であったかを理解することは、複数の DIY DevOps チャレンジャーと対面することが多いため、難しい場合があります

#### FY2024 戦略

FY 2024 の過程で 60 のインタビューを Clozd と行います。私たちの目的は、これらのほぼ過半数を勝利ではなく敗北とすることです。これらは、会社としての私たちの欠点について、より大きな洞察を与える傾向があるからです。また、GitLab セールスに沿って地理的にインタビューを分割します。

私たちの勝利基準:

- 大きいほど良い（理想的にはエンタープライズ顧客）
- 最近 GitLab を選択し、正常にオンボーディングされた顧客
- 他の競合相手（できれば GitHub または他の主要な競合相手）を評価していた顧客
- 「興味深い」基準（例: 私たちの強みであるか、通常競合相手に負ける要素であって、単に私たちが安かったからではない）で私たちを選択した顧客

私たちの敗北基準:

- 最近 GitLab の競合相手を選択した顧客
- 大きいほど良い（理想的にはエンタープライズ顧客）
- 最終決定で私たちを検討した顧客（例: 決定が GitLab vs GitHub に絞られた）
- 理想的には、敗北の理由がまだ完全には明確でない敗北（例: 私たちが競合相手との違いを伝えたが、彼らは納得しなかった）。
- 価格設定だけに帰着しなかった敗北

#### Clozd プロセスとリスク軽減計画

勝敗のコミュニケーションに関連するリスクを軽減するために、以下の計画がプライバシーチームと内部で合意されています:

| プロセス |
| ------ |
| Clozd は、米国がオプトアウトであるため、見込み客/リードからのさらなる同意なしに米国で勝敗のコミュニケーションを送信できます。 |

| Clozd は、見込み客/リードと Win 機会がある場合、世界の他の地域で自動化されたコミュニケーションを送信できます。なぜなら、将来のコミュニケーションは取引の対象となり、ソフトオプトインは世界の大部分で利用可能であるからです |

| 一般的に、Clozd は、見込み客/リードと敗北がある場合、世界の他の地域で自動化されたコミュニケーションを送信できません。なぜなら、その場合、ソフトオプトインは利用できないからです。ただし、以下が真実であれば、これらのコミュニケーションを送信できます: |

| GitLab のセールス担当者が会話中に見込み客/リードに、敗北分析のために Clozd によって連絡を取ることができるかどうかを尋ね、見込み客/リードが同意した |

| その同意のある種の証拠（例: メール確認や、見込み客/リードが後続のコミュニケーションに同意したと述べる同時期のコールメモ）がある |

| Clozd ビジネスオーナーである @jkempton は、前述の 2 つのサブ箇条書きを検証した上で、連絡先情報を Clozd に手動で送信します |

## 比較プロセス

GitLab では、誰もが競合インテリジェンスプロセスに貢献できます。私たちは以下の目標を達成することを目指しています。

1. **公正**: いかなる CI プロセスも信頼できるためには公正である必要があります。
2. **正確**: 私たちは、Web サイト、製品ドキュメント、デモビデオ、許可されている場合のトライアル、顧客からの情報、さらには競合相手からの入力を取得することによって、可能な限り正確であることを目指しています。例えば、最近、競合相手のプロダクトマネージャーが、すぐに行った分析の修正について私たちに連絡してきました。
3. **簡単に利用可能**: 私たちは、Web サイトで情報を利用可能にすることによって、情報を簡単に利用可能にしています。

## 機能が重複する競合他社と製品

GitLab は **[DevOps ツール](https://about.gitlab.com/why-gitlab/)** のエコシステムに存在し、これらのツールのいずれかと相互作用する必要があるかもしれません。多くは重複する機能を持っていますが、それは私たちが必ずしもそれらと直接競合することを意味しません。ユーザーは、**[エンドツーエンド DevOps のための単一アプリケーション](https://about.gitlab.com)** として GitLab に組み込まれているすべての機能を取得するために、このリストから複数のソリューションをパッチで組み合わせる必要があります。

私たちは、これらの製品を DevOps ツール比較ページにも含める傾向があり、顧客が私たちが必ずしも競争的観点ではなく、完全な風景をどのように見ているかを包括的に理解できるようにします。GitLab が誰と競合するかについての詳細は、このハンドブックページ[GitLab が誰と競合するか](/handbook/product/ux/competitor-evaluations/) を参照してください。

## 顧客の声

いつものように、これにおいて顧客の声は絶対に重要です。幸いなことに、GitLab を含む DevOps ツールについての顧客フィードバックをキャプチャする、いくつかのそのようなサードパーティの中立サイトがあります。GitLab に関する顧客の声でフィードバックを持つサイトはこちらです。

- [G2](https://www.g2.com/products/gitlab/reviews)
- [TrustRadius](https://www.trustradius.com/products/gitlab/reviews)

## その他の比較サイト

深いレベルでの比較は困難で時間がかかります。DevOps ツールを比較する誠実で強力な努力をしてきた他のいくつかのサイトを認めたいと思います。これらの一部は競合相手によるものですが、GitLab の透明性の価値に従い、見込み客に包括的な絵を提供するために、ここにあります:

- [XebiaLabs](https://digital.ai/learn/devsecops-periodic-table/)
- [CNCF](https://landscape.cncf.io/)
- [Gitea](https://docs.gitea.io/en-us/comparison/).

## 私たちが向かう先

- 既存の競合カードの作業に基づいて、プラットフォームと DIY DevOps の競合相手の両方から、また AI などの主要なトレンドのトピックに関する、内部の競合コンテンツを拡大しています。より機能ごとの比較については、私たちは、情報を可能な限り最新に保つために、製品チームの内部知識を活用します
- 勝敗データへの洞察を改善し、それに応じて戦略的ガイダンスを強化する
- マーケティング戦略に沿って、外部向けの競合コンテンツを再フォーカスする。

### 戦略

競合インテリジェンスチームの目標は、セールスチームが競争して勝つのを可能にする完全なアセットセットとその他の納品物を提供することです。
以下の図は、全体的なアプローチをキャプチャしています。考え方は、各ユースケース-競合相手について、戦略ドキュメント (Crayon でホストされている) を除いて、他のすべてのリソースは一般的に公開でアクセス可能であるべきです。これは GitLab の透明性の価値に従っています。
