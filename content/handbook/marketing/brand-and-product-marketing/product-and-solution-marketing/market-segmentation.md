## 業界バーティカル

さまざまなバーティカルや業界は、それぞれ固有の市場に対応する中で特有の課題に直面しています。コンプライアンスや規制圧力から、特定の市場混乱まで、これらのバーティカルの IT チームは独自の課題に対するソリューションを開発しています。

- 金融サービス
- 自動車
- ヘルスケア
- 連邦／政府／公共セクター
- 小売
- 高度製造（自動車とは別）
- 石油＆ガス／エネルギー

 セールスセグメンテーションに関する詳細情報は [Go To Market Resources page](/handbook/sales/field-operations/gtm-resources/) を参照してください。

### 金融サービス

 | 課題 | GitLab はどう貢献するか |
 | --------- | ------------ |
 | **セキュリティ** | GitLab の単一プラットフォームは SDLC ツールチェーンへのアクセスを制御・管理し、開発プロセスへの信頼性を高めることに貢献します。 開発ライフサイクルのコアな一部としてセキュリティスキャンを監査、レビュー、自動化する能力により、構築されるすべてのアプリケーションの安全性向上にさらに貢献します。 |
 | **老朽化したインフラストラクチャ** | GitLab はモダンなクラウドネイティブ（kubernetes）への移行を簡素化します |
 | **コンプライアンス** | SDLC ツールチェーンへのアクセスを制御・管理する単一のプラットフォーム。 組み込みの監査、レビュー、自動化により信頼できるパイプラインの作成に貢献します。 |
 | **新しいテクノロジー（ビジネス混乱／デジタルトランスフォーメーション）** | これは、Transportation as a Service プラットフォームとインフラを構築するソフトウェアチームに、迅速かつ応答性の高い設計、開発、イテレーションを要求します。 GitLab と Concurrent DevOps は、チームがソフトウェア出荷のサイクルタイムを劇的に削減することを支援します。 |

### 自動車

| 課題 | GitLab はどう貢献するか |
| --------- | ------------ |
| **コネクテッドカー（OTA アップデート）:** 車両がモバイルコンピュータになるにつれて、車載ソフトウェアの更新の重要性とコストが重要な課題となります。 ディーラーで更新をインストールしてもらうのは高コストで支障も大きいため、業界は OTA 更新を可能にするアプローチへ移行しています。 | OTA 更新を支援するため、自動車メーカーとサプライヤーは、ソフトウェア変更の開発、テスト、さらにはデプロイを合理化する迅速で効率的なソフトウェアデリバリーツールチェーンに依存します。 | 
| **Transportation as a Service** | この市場の変革では、Transportation as a Service を可能にするテクノロジープラットフォームの品質、シンプルさ、採用が差別化要因となります。これは、Transportation as a Service プラットフォームとインフラを構築するソフトウェアチームに、迅速かつ応答性の高い設計、開発、イテレーションを要求します。（GitLab／Concurrent DevOps） |
| **プラットフォーム統合** | 製造の複雑さと無駄を削減するため、プラットフォーム統合により、メーカーは基盤となるテクノロジーと消費者向けサービス（ソフトウェア）で自社製品を差別化することになります。ソフトウェアが差別化要因となり、急速に変化する市場に対応するために迅速なイノベーションが必要となります。 |

### ヘルスケア

| 課題 | GitLab はどう貢献するか |
| --------- | ------------ |
| **セキュリティ＆データ保護** ヘルスケアではシステムとデータの安全性確保が不可欠です。これは開発から本番までヘルスケアシステムのライフサイクル全体にわたる課題で、ほぼすべてのヘルスケア IT リーダーにとって最優先事項です。 | SDLC ツールチェーンへのアクセスを制御・管理する単一のプラットフォーム。 組み込みの監査、レビュー、自動化により信頼できるパイプラインの作成に貢献します。 |
| **患者体験** ヘルスケア市場は、患者体験の改善を中心に大きく変革されており、患者が自身のケアに参加しやすくしています。モバイルアプリケーションから電子健康記録、その他のイノベーションに至るまで、ヘルスケア IT リーダーは患者体験を向上させるためにデリバリーとイテレーションを加速しようとしています。 | 迅速なデリバリーとイテレーションが患者体験向上の特徴です。 UX のイノベーションは、IT チームが迅速に学び革新できる場合にのみ可能です。 Concurrent DevOps と GitLab は、デリバリーを合理化し、IT チームが患者体験の向上に集中できるようにすることに貢献します。 |
| **HIPAA** | SDLC ツールチェーンへのアクセスを制御・管理する単一のプラットフォーム。 組み込みの監査、レビュー、自動化により信頼できるパイプラインの作成に貢献します。 |
