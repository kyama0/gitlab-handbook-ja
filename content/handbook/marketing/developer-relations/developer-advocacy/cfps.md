## CFP リソース

イベントの講演募集 (Call for Paper) の応募フォームはさまざまです。1000 文字のアブストラクトを求めるものもあれば、短めの経歴を好むもの、講演で学べることの記入を求めるものもあります。Developer Advocacy チームは CFP のアブストラクトの管理に Google docs を使用しており、@dnsmichi は経歴とヘッドショットの URL の SSoT としても doc を使用しています。

## CFP（Call for Proposals）の管理方法

### 私たちのイベントリスト

毎年、developer advocacy は、カンファレンスの講演募集 (CFP) プロセスを実施する、私たちのエコシステムにおける主要なイベントに優先順位を付けます。ライトニングトーク、CFP 応募などのイベントへの関与は Issue で管理され、それらは[年次戦略エピック](https://gitlab.com/groups/gitlab-com/marketing/developer-relations/-/epics?state=opened&page=2&sort=start_date_desc&label_name[]=DevRel-Events&label_name[]=Events)にリンク・要約されています。私たちのイベントのカレンダービューも以下でご覧いただけます。

## Developer Advocate に CFP の応募を依頼する

あなたのイベントへの CFP 応募を Developer Advocate に依頼するには、次の手順を実行してください。

1. [新しい Issue を開きます](https://gitlab.com/gitlab-com/marketing/developer-relations/developer-advocacy/developer-advocacy-meta/-/issues/new?issuable_template=developer-advocate-request)。
1. `developer-advocate-request` テンプレートを使用し、`External Request` セクションに記入してください。
1. Issue に `CFP` ラベルを付けてください。

### CFP の管理

参加する CFP プロセスごとに、イベントの詳細、CFP 応募の要件、その他、潜在的な登壇者に役立つ可能性のある関連情報を記載した [CFP Issue](https://gitlab.com/gitlab-com/marketing/developer-relations/developer-advocacy/developer-advocacy-meta/-/issues/new?issuable_template=cfp-meta) を作成します。Issue テンプレートには、各 CFP のすべての応募を確実に追跡するために必要な手順のガイドが含まれています。

#### Issue ワークフロー

すべての応募、採択された講演、フィードバック、その他の結果の追跡は CFP Issue で行われます。CFP ワークフローは[スコープ付きラベル](/handbook/marketing/developer-relations/developer-advocacy/workflow/#cfp-labels)を使用して Issue をさまざまなステージに遷移させます。必要なラベルを適切に適用するには、[CFP ワークフローのハンドブック](/handbook/marketing/developer-relations/developer-advocacy/workflow/#cfp-workflow)に従ってください。

`CFP` ラベルが付いたすべての Issue は、オープン・応募済み・採択済みの CFP の概要を提供する専用の [CFP Issue ボード](https://gitlab.com/groups/gitlab-com/-/boards/9701731?label_name%5B%5D=developer-advocacy&label_name%5B%5D=CFP)に表示されます。

## CFP 応募のためのヒント

優れたカンファレンスの講演提案を書くことは、アートでありサイエンスでもあります。サイエンスは技術的なコンテンツの詳細です。アートは、それを聴衆 — カンファレンス委員会 — に訴える形で提示することです。

CFP を書く際に踏むべき手順をいくつか紹介します。

1. **イベントをリサーチする**: イベントの Web サイト、特に講演提案のページを詳細に読み込みます。読み進めながら、資料全体を通して繰り返し現れる主要なトピックやテーマに注目してください。
2. **推奨されるトピックを把握する**: イベントがテーマやトピックを提案している場合、講演提案をそれに合わせるのが最善策です。
3. **カンファレンス委員会について知る**: オンラインの資料と同じくらい、人々も重要です。委員会を調べて、彼らが情熱を注いでいるトピックを見つけられるか確認してください。過去の登壇者や、その他カンファレンスに関与している人々もリサーチすると、彼らがどのようなトピックや登壇者を本当に求めているのかを理解するのに役立ちます。
4. **仮のタイトル、テーマ、アウトラインを書き出す**: 仮のタイトルから始めると、白紙症候群を克服するのに役立ちます。そこから、テーマをブレインストーミングします。あなたは何を教えようとしているのですか? その物語の SCQA（Situation, Complication, Question, Answer）を 1 行で説明するとしたら、それは何でしょうか。そして SCQA のアウトラインを埋めていきます。
5. **助けを求める**: 主題に詳しい友人や同僚に提案をレビューしてもらいましょう。手直しし、繰り返し、そして応募しましょう!
