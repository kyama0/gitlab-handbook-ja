パートナープログラムハンドブックの本セクションでは、GitLab パートナーのテクニカル担当の視点から、アセットとイネーブルメントを確認します。

パートナーハンドブックページに加えて、以下に概観するセールスガイド、ユースケース、トレーニング資料、プログラムガイドがあります。パートナー固有のコンテンツは [GitLab Partner Portal](https://partners.gitlab.com/) にホストされています。これらの資料は GitLab で効果的に販売、サービス提供し、目標を達成するための優れた出発点となります。

以下のリンクの一部は、先に [GitLab の Partner Portal](https://partners.gitlab.com/) にログインする必要があることに注意してください。まだ未登録の場合は[こちらからポータルアクセスを登録](https://partners.gitlab.com/partner/registration)してください。

ステップバイステップのオンボーディングガイドについては、[GitLab POWER ページ](/handbook/resellers/partner-enablement/power)を参照してください。

これらの情報をすべて検索するのに役立てるため、[LLM を使うこと](/handbook/resellers/partner-enablement/llm-system-prompts)を検討してください。

  - Release Overview Web サイトの [Deprecations by Version ページ](https://gitlab-com.gitlab.io/cs-tools/gitlab-cs-tools/what-is-new-since/?tab=deprecations) と同じデータを別の形式で提供します。
- [GitLab Releases Blog](https://about.gitlab.com/releases/categories/releases/)
  - Release Overview Web サイトと同じ情報ですが、顧客と共有しやすい、整形済みのブログ記事になっています。
- [The GitLab Project](https://gitlab.com/gitlab-org/gitlab)
  - GitLab ソフトウェア自体のソースコードです。最も有用なのは [Issues ページ](https://gitlab.com/gitlab-org/gitlab/-/issues) で、誰でも機能やバグの状態を確認し、計画中か取り組み中かを確認できます。
