## 概要

Okta は SaaS アプリケーションとクラウドプラットフォーム向けの ID およびシングルサインオンソリューションで、GitLab ではほとんどのアプリケーションへのアクセスに使用されています。これにより、私たちは日常的に使用するアプリケーションへの認証と認可を単一のダッシュボードに統合し、すべての GitLab チームメンバーに対して一貫性があり、安全で監査可能なログイン体験を確保できます。

## メリット

- **アプリケーションオーナー**
  - 自動化されたユーザープロビジョニングとグループ管理
  - 認証情報をユーザーに開示せずに Web アプリケーションの共有認証情報を透過的に管理する機能
  - ユーザーへの集中化されたアクセスにより、アプリケーションプロファイルの追加、削除、変更を、すべてのユーザーを更新する必要なく簡単に行えます。
- **ビジネスセキュリティ上のメリット**
  - 当社の資産に対するゼロトラストベースの認証制御を有効化し、より高い確実性をもって主要資産への認可された接続を許可できます。
  - 当社の Tech Stack 内で使用している 80 以上 (増加中) のクラウドアプリケーションへのログインプロセスを管理します。
  - 自動化と HRIS システムへの統合により、これらアプリケーションにアクセスするユーザーのプロビジョニングおよびデプロビジョニングプロセスを管理します。
  - 主要資産への認証要件について信頼およびリスクに基づいた判断を行い、一貫したユーザー体験を確保するためにそれらを適応させます。
- **ユーザーのメリット**
  - 必要なすべてのアプリケーションを一箇所にまとめた単一のダッシュボード。
  - ログインパターンを学習・適応する管理された SSO と多要素認証により、必要なアプリケーションへのアクセスをよりシンプルにします。
  - フレンドリーなユーザー体験を備えた透明性のあるセキュリティ制御。
  - 改善された 2FA ユーザー体験。
