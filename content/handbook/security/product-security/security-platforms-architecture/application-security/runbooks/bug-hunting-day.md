## バグハンティングデーのプロセス

アプリケーションセキュリティチームは、毎月最終金曜日にバグハンティングデーを行っています。

### 目標

目的は、チームメンバーがレビュー中に観察、議論、または気付いたが、詳細なレビューやテストに入る時間がなかったものについて調査する時間を持つことです。

バグハンティングデーは必須ではありません。特に忙しい月もあるかもしれず、そのような月にはバグハンティングデーを行わない場合もあります。

### どうやって始めるのか？

1. `bug-hunting-day` テンプレートで Issue を開くか、[こちら](https://gitlab.com/gitlab-com/gl-security/product-security/appsec/appsec-team/-/issues/new?issuable_template=bug-hunting-day) をクリックして新しいものを作成します（テンプレートはすでに選択されています）。
1. 未解決のアイデアについて以前の Issue を確認します
1. アイデアを Issue に書き込むか、思いつくものがなければチームにトピックを尋ねます
1. 当日、時々 Zoom ミーティングに参加して他のチームメンバーと交流します。
