## 目的

このガイドの目的は、AnswerBase の品質に対する期待を確立することです。

## 品質に対する期待

### Answers (回答)

AnswerBase の回答は、以下の特徴を持つべきです。

- 単なる `yes` または `no` の回答以上のもの
  - 回答にはサポートとなる説明が含まれているべきです。
    - サポートとなる説明は、なぜ私たちが質問に対して `yes` または `no` と回答しているのかを正確に説明するものでなければなりません。
    - サポートとなる説明は、`yes` または `no` の回答を正当化するコントロール、製品機能、ポリシー/手順などに立ち返って参照するものでなければなりません。
      - 私たちは、アンケートで顧客にサポートとなる説明を提供する義務はありません。
      - サポートとなる説明は、回答でカバーされるトピックに対するすべてのチームメンバーの理解を強化します。
      - サポートとなる説明は、AnswerBase をすべてのチームメンバー向けの真の知識ベースとして確立するのに役立ちます。
  - 回答は少なくとも年に 1 回、正確性についてレビューされます。

### Tags (タグ)

- 回答にはタグが割り当てられているべきです。
  - タグはチームメンバーがより迅速かつスムーズに検索できるようサポートします。
  - タグは少なくとも年に 1 回、正確性についてレビューされます。

### Libraries (ライブラリー)

- ライブラリーには適切な名前と説明が割り当てられているべきです。
  - 名前は、各ライブラリーに含まれる回答の性質を説明するものでなければなりません。
  - 説明は、すべてのチームメンバーがライブラリーに入ったときに見ているものを正確に理解できるよう、各ライブラリーに含まれる回答の性質をさらに説明するものでなければなりません。
  - コンテンツが入っていないライブラリーには、ライブラリー名に `(COMING SOON)` を付加するべきです。
  - ライブラリーオーナーは、ライブラリーの維持を担当する SME を表すべきです。
  - ライブラリーの存在の正当性は、年に 1 回評価されるべきです。
  - ライブラリーは、その存在が正当化されなくなった時点で廃止するべきです。
    - ライブラリーには廃止日があり、ライブラリーの説明を通じて伝達されるべきです。例:
        `***This library is set for decommissioning on YYYY/MM/DD*** Description of Library`
    - 廃止日は、ライブラリーが正当化されないと判断されてから 30 日後とするべきです。
