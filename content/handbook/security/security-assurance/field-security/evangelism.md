## なぜエバンジェリズムを行うのか?

Field Security の重要なミッションの 1 つは、GitLab のセキュリティ慣行と私たちのプラットフォームのセキュリティに対する内外の認知を向上させることです。私たちが反復し成長を続けるためには、見込み顧客や顧客と意義ある形で関わり影響を与える能力を向上させなければなりません。

Security エバンジェリズムにより、私たちは見込み顧客や顧客の「ニーズ」と「ウォンツ」をより理解できるようになり、彼らに価値ある製品やサービスを提供する立場をより良く築くことができます。

## どのようにエバンジェリズムを行うのか?

Field Security の目標は、Security エバンジェリズムに包括的なアプローチをとることです。私たちはダイナミックでありたいと考え、情報の収集と共有の伝統的な方法を超越したいと考えています。

### 文書による関与

- GitLab の製品やサービスについて特定された懸念事項や改善領域をドキュメント化する年次の[Field Security Study](/handbook/security/security-assurance/field-security/field-security-study/)。

## あなたはどのように協力できるか?

すべてのチームメンバーからの継続的でオープンなフィードバックは何より歓迎されます。見込み顧客や顧客と接していて、懸念事項、提案、「ウォンツ」、「ニーズ」を学んだ場合は、その情報を Field Security に転送してください。あなたのフィードバックは、私たちが GitLab がより多くの価値を顧客と業界に提供できるよう助けるために、私たちのロードマップと戦略を推進する力となります。
