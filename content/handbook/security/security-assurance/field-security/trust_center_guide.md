## GitLab は顧客のニーズに合わせた Trust Center を提供しています

- **GitLab.com ページ**:
このページは、GitLab.com SaaS およびセルフマネージドの見込み顧客と既存顧客に、ベンダーセキュリティ評価を完了するための詳細なセキュリティ情報を提供します。

- **GitLab Dedicated ページ**:
このページは、GitLab Dedicated の見込み顧客と既存顧客に、ベンダーセキュリティ評価を完了するための詳細なセキュリティ情報を提供します。

## Trust Center ドキュメントの2種類

- **すぐに利用できる情報**: トラストジャーニーの第一歩であり、これらのドキュメントは公開されており、GitLab のセキュリティに対するアプローチを紹介するために設計されています。[Trust Center](https://trust.gitlab.com) から直接利用できます。

- **NDA下の情報**: これらのドキュメントは、ベンダーセキュリティ評価を完了するために、見込み顧客と既存顧客に詳細なセキュリティ情報を提供します。ドキュメントの機密性により、特定のドキュメント（SOC 2 Type 2 など）を閲覧する前に NDA の締結が必要です。NDA の署名プロセスは、Trust Center を通じて直接行われます。

GitLab のセキュリティレポートの性質上、[Trust Center](https://trust.gitlab.com) の特定のドキュメントは、Trust Center のクリックラッププロセスを通じて NDA に署名した後にのみ利用可能となります。

#### SME 所有のコラテラル

Field Security 外の SME が所有するコラテラルについては、年次レビューと保守セッションを実施しています。SME には GitLab の Issue で通知され、レビューと保守のために関連するコラテラル要素が割り当てられます。完了後、更新されたコラテラル要素は適切な CAP に追加され、公開のために準備されます。

### 追加サポート

さらに質問がある場合は、以下のステップに従ってください。

- 見込み顧客の方: [リクエストフォーム](https://about.gitlab.com/sales/) にご記入いただければ、担当者からご連絡いたします。
- 現在の顧客の方: GitLab の[アカウントオーナー](/handbook/sales/#initial-account-owner---based-on-segment) にご連絡ください。アカウントオーナーが分からない場合は、[セールスにお問い合わせ](https://about.gitlab.com/sales/)いただき、アカウントオーナーへの接続をリクエストしてください。
- GitLab チームメンバーの方: [#sec-fieldsecurity](https://gitlab.slack.com/archives/CV5A53V70) Slack チャンネルの **Customer Assurance ワークフロー** を使用して、Field Security チームに連絡してください。
