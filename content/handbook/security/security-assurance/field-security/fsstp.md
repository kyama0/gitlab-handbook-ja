## なぜセキュリティのセールストレーニングプログラムなのか？

Field Security の重要なミッションのひとつは、GitLab のセキュリティプラクティスとプラットフォームのセキュリティに関する社内外の認知を高めることです。私たちがイテレーションを重ねて成長を続けるにつれ、見込み客や顧客と意義ある形で関わり、影響を与える能力を向上させていく必要があります。

セールス組織はこの取り組みの中心的な存在です。彼らは GitLab を販売し、顧客や見込み客に対する信頼されるアドバイザーとなる重要な役割を担っています。セキュリティの観点では、この役割は以下の2つの側面で重要です。

- 最初の DevSecOps プラットフォームとして、セキュリティは見込み客が GitLab を選び、開発プロセスにセキュリティをネイティブに統合したいと考える中心的な理由です
- 企業のサプライチェーンが複雑化するにつれ、セキュリティのステークホルダーがセールスプロセスの中心となることが多くなっています。したがって、社内セキュリティプログラムの成熟度を示すことは、大きな競争上の優位性となります

セキュリティセールスイネーブルメントプログラムの目標は、次のセクションで詳述するアプローチを用いてこれらの目的を達成することに注力することです。

## 5つの目的

### セキュリティを早期に組み込む

> セールスのステークホルダーが顧客/見込み客とセキュリティについて積極的に関わるのを支援する

### セキュリティ業務の合理化

> 顧客のセキュリティレビュープロセスに関連する管理上の負担を軽減する

### セキュリティの役割を理解する

> 顧客向けアシュアランス活動と Field Security の業務に関する理解を強化する

### セキュリティをよりよく位置付ける支援

> セールスがセキュリティドキュメントとリソースを位置付け、サイクルを短縮できるよう導く

### セキュリティへの連絡方法

> 質問・サポート・エスカレーションのためにどこへ行き、誰に連絡すべきかを知る

## 次世代のセールスセキュリティトレーニング

私たちの目標は、GitLab の[Field Security](../field-security/) プログラムの次世代を構築することです。このプログラム強化の一環として、セキュリティセールストレーニング・イネーブルメントプログラムを確立します。以下の取り組みを先導していきます。

- **GitLab Security on Demand** — セールスセキュリティイネーブルメントの Netflix
- **ジャストインタイムのマイクロイネーブルメント** — 必要なときに必要なことを学ぶ
- **AI 搭載 AnswerBase** — セキュリティに関する会話のコンパニオン
- **顧客にフォーカスしたセキュリティイネーブルメントセッション** — 見込み客が関心のあるトピックでの習熟度を構築
- **統合されたフィードバックプロセス** — ニーズに合わせて常にプログラムを改善できるように
- **ハンドブックファーストの包括的なドキュメント** — 必要な情報がハンドブックページひとつ先に
