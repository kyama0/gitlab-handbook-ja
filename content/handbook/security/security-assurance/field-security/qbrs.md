## 概要

GitLab は、毎四半期の最初の月に[四半期ビジネスレビュー（QBR）](/handbook/sales/qbrs/)を実施し、前四半期のパフォーマンスや学びの振り返り、フォーキャストのレビュー、そして将来のセールス活動を支援するリクエストの優先順位付けを行います。透明性とコラボレーションの精神に則り、Field Security は QBR プロセスに参加し、将来のセキュリティ関連の懸念事項に関する情報や推奨事項を収集します。

## プロセス

1. [Field Manager QBR ロールアウトスケジュール](/handbook/sales/qbrs/#sample-field-manager-qbr-roll-out---virtual)の一環として、Field EBA が Field Security チームにカレンダー招待を送信します。空き状況に応じて、チームメンバーが各 QBR セッションに参加します。
1. Field Security チームメンバーは、セキュリティ関連の懸念や問題を取り入れます。
1. Field Security チームメンバーは、それらの懸念事項を評価し、[GitLab の観察事項管理](/handbook/security/security-assurance/observation-management-procedure/#risk-ratings)手順を使用してレーティングします。
1. サマリ Issue が、検討のために関連するステークホルダーに共有されます。必要に応じて、セキュリティ上の懸念は[年次 Field Security 調査](/handbook/security/security-assurance/field-security/field-security-study/)に追加される場合があります。
