## GitLab Certified Migration Services Engineer

GitLab のマイグレーションサービスの提供キャパシティをスケールできるよう、[GitLab プロフェッショナルサービス](https://about.gitlab.com/services/catalog/) は、このサービスを顧客にデリバリーしたいエンジニア向けに認定プロセスを提供しています。このプログラムは現在、GitLab チームメンバーおよび選定された [GitLab Certified Services Partners](/handbook/resellers/services/) が利用できます。

このプログラムは、顧客のデータをレガシーシステムから GitLab に正常にマイグレートするために必要なスキルと知識を開発し検証するための体系的なアプローチを提供します。

## 検証されるスキルと知識

認定マイグレーションエンジニアは、次の目標を正常に達成できます:

- GitLab UI 内または API 経由で利用可能な GitLab の機能を使用して、GitLab、GitHub、Bitbucket などの一般的なソースコード管理（SCM）データソースとの間でデータをインポート／エクスポートできる。
- git エンベロープに格納されたデータと GitLab（またはその他の SCM）プロジェクトに格納されたデータ要素に関連する一般的なマイグレーション問題のトラブルシューティングができる。
- 与えられたプロジェクトの Statement of Work（SOW）または類似のスコーピングドキュメントの内容を解釈し、スコープ内またはスコープ外のタスクを特定できる。
- マイグレーションプロジェクトのスコーピングドキュメントが与えられたら、顧客とディスカバリーミーティングを実施することでスコープを検証および洗練できる。
- プロフェッショナルサービス（PS）のマイグレーション自動化ツールである Congregate を使用して、自動化された SCM マイグレーションを完了できる。
- 顧客のマイグレーションに関する問題のトラブルシューティングやアプローチのレビューについて、PS マイグレーションチームからサポートを依頼するための正しいガイドラインとプロセスに従うことができる。

### 候補者プロセス

GitLab Certified Migration Services Engineer 認定を取得するために必要なステップは次のとおりです。

**ステップ 1**: GitLab チームメンバーであればマネージャーに、パートナーであれば GitLab 担当者に連絡し、この認定の取得に対する関心について話し合います。

**ステップ 2**: GitLab 担当者から提供されたネームスペースに、[migration-onboarding.md issue テンプレート](https://gitlab.com/-/ide/project/gitlab-com/www-gitlab-com/tree/master/-/sites/handbook/source/handbook/customer-success/professional-services-engineering/gitlab-certified-migration-services-engineer/) を使用して新しい Issue を作成します。パートナーの場合は、ネームスペースへのアクセスを得るため、または Issue の作成に支援が必要な場合は、GitLab 担当者に連絡してください。

**ステップ 3**: Issue 説明の Candidate Tasks セクションに記載された各項目を完了します。タスクを進めながら、Issue に記載された GitLab プロジェクトコーディネーターに連絡し、シャドウイングおよび最初のリードエンゲージメントをスケジュールしてください。完了する必要がある主要なタスクは次のとおりです。

- マイグレーションサービスのリソース（ドキュメンテーションおよびデリバリーキットを含む）をレビューする
- 採点付きマイグレーションアセスメントに合格する
- congregate-onboarding ワークショップを完了する
- PSE マイグレーションエンジニアが主導するマイグレーションをシャドウする
- PSE マイグレーションエンジニアが観察してフィードバックを提供する中で、マイグレーションをリードする

**ステップ 4**: 認定を取得すると、デジタルベリフィケーションバッジを受け取ります - これを LinkedIn プロフィールに追加してください！
