1. [更新版 GitHub vs. GitLab 比較ページ](https://web.archive.org/web/20240331181506/https://about.gitlab.com/competition/github/) - 比較インフォグラフィックと GitHub のギャップ・影響の完全リストが含まれます。
1. [GitHub License Competitive Update](https://docs.google.com/presentation/d/1xVwIBrsrEIKjfdev9E4T7YaSQT_LVtH_kuex7wqAA_U/edit#slide=id.g29a70c6c35_0_68) - 今回の発表による GitHub のライセンス変更の詳細解説。
1. [新しい GitHub-GitLab 比較サマリー](https://docs.google.com/presentation/d/1WAim4zkwaurifcaJn3bZp31CIzzf8AhnGTcbIdXpug8/edit#slide=id.g6fc8bb1d32_0_190) - GitHub に欠けている GitLab のケイパビリティ。
1. [GitHub CI/CD 分析](https://docs.google.com/presentation/d/1fDzXpgmvKyLpT0QTsGFnisFUeceNPsaL66vRVJ-sdpg/edit#slide=id.g730aeab9dc_0_262) - GitLab と GitHub が特定の CI/CD ケイパビリティをどのように提供しているかの比較。

または、Competitive Intelligence チームによる上記リソースの概要については、[Field Enablement ウェビナーの録画](https://www.youtube.com/watch?v=1iMBhe0wUdI&feature=youtu.be) をご覧ください。質問は Slack の #competition へ。

### GitLab の最新情報 🚀

GitLab 12.10 リリースが公開されました！ お客様や見込み客向けの要点は以下のとおりです:

- セキュリティとコンプライアンスのリスクを低減: GitLab 12.10 には、Requirements Management の最初の MVC が含まれており、これは要件からコードまでトレーサビリティのある本格的な要件管理に向けた最初のステップです。また、プロジェクト向けの Compliance Framework Labels、HIPAA 監査プロジェクトテンプレート、コンプライアンスダッシュボードへの改善など、主要なコンプライアンス機能も提供しました。これらすべてが、コンプライアンスを犠牲にせずに、より速く、より効率的に提供することを支援します。
- 運用効率の向上: 企業は Issue のヘルスステータスを追跡・共有することで複数のプロジェクトをより簡単に管理でき、Jira Issue インポーターを活用することで、ツール間の切り替えに費やす時間を減らし、優れたソフトウェアの構築に集中する時間を増やせます。
- より良い製品をより速く提供: 組織は GitLab の AWS Fargate でのオートスケーリング機能を使用してランナーを自動スケーリングし、デリバリーを加速し、パイプラインが需要に応じて効率的にスケールできるようにします。

[詳細は 12.10 リリース投稿をご覧ください](https://about.gitlab.com/releases/2020/04/22/gitlab-12-10-released/)
