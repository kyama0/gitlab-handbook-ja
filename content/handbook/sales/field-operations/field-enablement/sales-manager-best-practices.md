## 新規チームメンバーのオンボーディング

- オンボーディングを成功させることは、チームメンバーが成功への道を歩み始めるうえで重要な役割を果たします。新しいチームメンバーが入社する前に、オンボーディングプロセスを確認し、何を期待すべきかを把握し、新入社員と期待値を擦り合わせられるよう準備してください。[オンボーディングページ](/handbook/people-group/general-onboarding/) には、[Sales 専用のオンボーディング Issue](/handbook/people-group/general-onboarding/#role-specific-templates) へのリンクなど、多くの有用なリソースが揃っています。
- オンボーディングプロセスについて詳しく知りたい場合は、2020 年 6 月 3 日に Sales リーダー陣と実施したオンボーディング教育セッションと AMA をご覧ください。

## アカウントプランのレビュー

- チームと定期的にアカウントプランをレビューしましょう。アカウントプランは四半期ごとにレビューすることが推奨されます。詳細は [Account Planning Coaching](/handbook/sales/account-planning/coaching-for-asms/) ページをご覧ください。

## Command Plan のレビュー

- この [Salesforce レポート](https://gitlab.my.salesforce.com/00O4M000004e1dZ) を使って、チームの Command Plan の集計を定期的にレビューしましょう。マクロとミクロの両面でチームの進捗を確認し、商談を妨げる可能性のある情報のギャップや障害を特定できます。

## 商談コーチング

- [Opportunity Consults Handbook ページ](/handbook/sales/command-of-the-message/opportunity-consults) をチェックしてください。
- Command of the Message と MEDDPPICC の [セールスマネージャー向け資料](https://internal.gitlab.com/handbook/sales/command-of-the-message/sales-manager-materials/)（社内限定）を定期的に活用してください。

## 戦略的コーチング

- 勝利を祝い、成功事例をチーム間に広めて、モチベーション向上に役立てましょう。
- 何がうまくいっているかについて、同僚のマネージャーからフィードバックや意見を引き続き活用しましょう。
- チームと 1:1 の担当者ディスカッションで「Why（なぜ）」から始めましょう（GitLab は組織全体で「Why」を説明することについて優れた成果を上げていますが、マネージャーは日々のやり取りでチームや個々の担当者向けに「Why」から始まるよう心がけてください）。
- パートナーセラーが Command of the Message と MEDDPPICC を活用して共同で商談を進められるよう支援しましょう。
- Command of the Message と MEDDPPICC のリフレッシュセッションを実施し、週に 1〜2 枚のスライド/ツールをカバーしましょう。
