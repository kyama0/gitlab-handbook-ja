## 概要

GitLab は Force Management の Command of the Message バリューベースのセールスメッセージング方法論を採用し、運用化しています。IP の制約により、このページから内容を GitLab の内部ハンドブックに移行しました。GitLab チームメンバーは [Command of the Message Internal GitLab ハンドブックページ](https://internal.gitlab.com/handbook/sales/command-of-the-message/) で追加のコンテンツとリソースにアクセスできます (注: GitLab チームメンバーで内部ハンドブックにアクセスできない場合は、[Okta](https://gitlab.okta.com/) 経由で GitLab Internal Handbook アプリにサインインしてください)。

## カスタマーバリュードライバー

バリュードライバーは、組織がプロアクティブに探しているか、必要としている可能性が高いものを記述し、GitLab が存在しなくても顧客のトップオブマインドのトピックとして存在します。バリュードライバーは買い手に裁量的な資金の再配分を促す可能性があり、バリューベースの顧客会話をサポートします。組織は以下のバリュードライバーのために GitLab を採用・実装します:

1. **オペレーショナル効率の向上** - *ソフトウェア開発ツールチェーンを簡素化して総保有コストを削減*
1. **より良い製品をより速く提供** - *ビジネス目標を達成するためにソフトウェア提供プロセスを加速*
1. **セキュリティとコンプライアンスリスクの削減** - *スピードを犠牲にすることなく、内部プロセス、コントロール、業界規制に準拠するためのプロセスを簡素化*
1. **クラウド導入と変革の加速** - *クラウドプロセスを標準化してビジネスアウトカムを加速*

## GitLab 差別化要因

| GitLab 差別化要因 | 説明 |
| ------ | ------ |
| **DevOps ライフサイクル全体のための単一アプリケーション** | - 単一アプリケーションとして提供される完全な DevSecOps プラットフォーム<br> - 1 つのインターフェイス、1 つのユーザーモデル、1 つのデータモデル<br> - DevOps ライフサイクル全体をサポートするソフトウェア「ファクトリー」<br> - 深く統合されており、開発者をより幸せでより効率的に<br> - 幅広い役割のための集約されたコラボレーション (待ち時間なしでコラボレーション) |
| **SDLC 全体にわたる AI 駆動ワークフロー** | - コードだけでなく SDLC 全体にわたる AI 駆動ワークフローにより、ソフトウェアをより速く作成し、非効率性を取り除き、迅速に反復 |
| **1 つのアプリケーションにおける主要な SCM と CI / CD** | - 1 つのアプリケーション内の DevOps ツールチェーンのバックボーン<br> - コードレビューとコラボレーションを合理化<br> - SCM または CI (またはその両方) から始められる<br> - 1 つのインターフェイス、1 つのユーザーモデル、1 つのデータモデル<br> - 他のツールと相互運用可能 (好きなツールを使い続けられる!)<br> - 実証済みのエンタープライズスケール |
| **組み込みのセキュリティとコンプライアンス** | - 標準提供のセキュリティ機能 (コードスキャン、依存関係スキャン、シークレット検出など)<br> - ポリシーコンプライアンスを促進する自動セキュリティテストと監査制御<br> - シフトレフト: 開発ライフサイクルの早期にセキュリティテストを移動。コードコミット時点でのセキュリティテスト<br> - 開発者は導入した新しい脆弱性について即座のフィードバックを得る<br> - 開発チームとセキュリティチームのより良いコラボレーションを実現<br> - DevSecOps のための 1 つのインターフェイス、1 つのユーザーモデル、1 つのデータモデル |
| **スケール対応の構築** | - GitLab はドキュメント化されたリファレンスアーキテクチャを使用して 5 万+ ユーザーまで水平スケール可能<br> - GitLab の単一 DevSecOps プラットフォームにより、組織は成長に伴うコストの増加方法を把握できる<br> - GitLab により組織全体を横断する計画立案が可能 |
| **ソフトウェアをどこにでもデプロイ** | - 任意の環境、任意のクラウドにデプロイ<br> - GCP、AWS、Azure、OpenShift、VMWare、On Prem、Bare Metal などをサポート<br> - ワークフローのポータビリティ: デプロイ先に関わらず 1 つのデプロイメントワークフロー |
| **Kubernetes 向けに最適化** | - マイクロサービスやサーバーレスなどのモダンなクラウドネイティブ開発パターンの導入を実現<br> - GitLab ユーザーインターフェイスからクラスターのセットアップと使用を可能にし、Kubernetes 学習曲線を最小化<br> - GitLab ユーザーインターフェイスから Kubernetes デプロイ済み Pod の詳細とログを表示・管理<br> - すべての Kubernetes デプロイ済みアプリケーションとそのデプロイ先クラスターの自動監視<br> - 自動カナリアメトリック収集と表示を備えた組み込みのカナリアおよびインクリメンタルデプロイメント戦略<br> - クラウドネイティブ技術の未来への積極的な貢献者 (Cloud Native Computing Foundation を通じて) |
| **エンドツーエンドのインサイトと組み込みのバリューストリーム管理** | - 共通データモデルにより DevSecOps ライフサイクル全体にわたるインサイトを独自に実現<br> - 時間経過に伴うワークアイテムのステータスを表示する設定可能なインサイトダッシュボード<br> - サイクル分析データがサイクルタイムの改善領域の特定に役立つ<br> - セキュリティインサイトが脆弱性のロールアップを提供<br> - プログラムレベルのインサイトがプロジェクトを軌道に保つのに役立つ |
| **柔軟な GitLab ホスティングオプション** | - クラウドで使用するか、自身の環境 (クラウドまたはオンプレ) にインストール<br> - GitLab.com と self-managed の機能パリティ<br> - ニーズが変わったときにあるホスティングオプションから別のものに移行<br> - スケーリングをサポートするハイブリッドホスティングモデルの実装 (クラウドの CI Runner など)<br> - どの環境でもアップグレードと保守が容易 |
| **迅速なイノベーション** | - 毎月のリリース、100+ ヶ月連続<br> - 新機能の絶え間ない流れ –> GitLab は時間とともにより価値が高まる<br> - 独自に透明な製品開発プロセス。顧客、パートナー、コミュニティが貢献可能 |
| **オープンソース、誰もが貢献可能** | - オープンコア開発モデルにより、誰もが製品機能に貢献可能<br> - 顧客、パートナー、コミュニティを巻き込む独自に透明な製品開発プロセス<br> - 強くて成長するコミュニティ (10 万+ の組織と数百万のユーザー)<br> - 製品体験内でのオープンソースイノベーションの活用 (例: Prometheus) |
| **コラボレーティブで透明性のあるカスタマーエクスペリエンス** | - 私たちは製品開発とコミュニティ・顧客とのエンゲージメントにおいてコアバリューを運用化<br> - ロードマップ、Issue、社内会議、プロセスに関する極度の透明性<br> - 私たちは GitLab を使って透明性のある形で顧客とコラボレーションし、Issue の解決とロードマップのイテレーションの両方を行う<br> - GitLab はスケールでの DevSecOps とアジャイル実践の進化するケーススタディを提供し、パブリックハンドブックを通じてアクセス可能 |
