## Ronnie の README

こんにちは、私は Ronnie です。GitLab の Support Engineering Manager の一員であることを誇りに思っています。

このページは、私と一緒に働くとどのような感じになるかを他の人に理解してもらうために作成したものです。特に、まだ私と一緒に働いたことがない方に向けています。

また、意図的に自分をオープンにすることで信頼関係を築く試みであり、チームに加わる人々の不安を軽減するために、良い職場関係についての考えを共有するものです。

## 私の助け方

* 私が話すのが速くなったら、遠慮なくゆっくり話すよう言ってください。または繰り返してください。
* チームの一員であれば、話す必要があるときは遠慮しないでください。私はあなたのためにいます。チームです。
* アイデア、コメント、提案は遠慮なく。いつでも歓迎します。
* ベストを尽くしつつ、必要なときに助けてくれる優秀な人たちがいることも覚えておいてください。
* あなたが何かのサービスで問題が起きたときに助けてもらいたいように、お客様にも同じように接してください。

## 私の仕事スタイル

マネージャーとして、チームは幸せで、自律が尊重され、信頼され、認められたときに最も良く機能すると思っています。

私の経験では、そこに到達する方法は人によって異なります。そのため、オープンな会話をし、意見を求め、達成可能な課題を与え、新しくやりたいことや学びたいことを聞くことが好きです。

## 他者に対して想定すること

* 最高の仕事ができない何かが起きていれば、話してくれるでしょう。
* 私たちが話していること以外にも、あなたには色々なことがあるはずです。
* 必要であれば GitLab の誰にでも助けを求めるでしょう。
* フィードバックを提供してくれるでしょう。
* まだ文書化されていない新しい知識はすべて文書化してくれるでしょう。

## 得たいもの

* チームからの信頼を得て、彼らが目標を達成できるよう支援できるようになること
* GitLab 製品についての知識。多く知れば知るほど、お客様をより良く支援できる
* コンフォートゾーンの外に出てチャレンジできる機会

## 私とのコミュニケーション

* Slack は素晴らしいので、そこで話しましょう。

  * チームに役立つ質問・コメントには #support_team-chat を使ってください。
  * 私とその他の人が必要なプライベートな会話には **プライベートチャンネル** を使ってください。
  * その他については @ralfaro にダイレクトメッセージしてください。

* 私は少し内向的なので、大きなグループ会話で静かにしていたら、質問やコメントでコンフォートゾーンから引き出してください。最終的には話しすぎてしまうかもしれませんが :)

  >
  > こんな感じで: Ronnie、Git でのリベースの理解に問題があります。

## 強み / 弱み

* やることに細部まで注意を払いすぎることがあります。たとえば写真を撮るとき、**傾いている**と感じることが多いですが、たった2度のことだったりします。これは改善する必要があります。

* 考えすぎることがあり、小さなことが頭の中で深さ N+K の N 分木になってしまいます。改善中です。

* 簡単にあきらめません。できるまで何度でも試み、学びます。

* 学ぶことをやめません。常にもう少し知れること、学べること、改善できることがあると信じています。
