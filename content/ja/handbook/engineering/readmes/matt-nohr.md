## Matt Nohr の README

GitLab の [AI Coding](/handbook/engineering/ai/ai-coding/) チームのエンジニアリングマネージャーです。

GitLab では以下のチームで以前働きました:

- AI Engineering: Code Creation
- Create: Code Review
- Create: Editor
- Create: Knowledge
- Ecosystem
- Monitor: Health
- Monitor: APM

リンク:

- [GitLab](https://gitlab.com/mnohr)
- [LinkedIn](https://www.linkedin.com/in/matthewnohr/)

### 働き方のスタイル

- [サーバントリーダーシップ](https://wheniwork.com/blog/the-ultimate-guide-to-the-servant-leadership-model)モデルを信じています
- 「走っているなら走らせておく。転んだら助け起こす」というような考え方をします
- [Defender（守護者）](https://www.16personalities.com/isfjs-at-work) 型のパーソナリティを持っています
- [行動へのバイアス](/handbook/values/#sts=bias%20for%20action) を好みます
- 「〜をお願いできますか？（Can you help with X?）」という依頼には良く反応します。「〜してください（Do X!）」という指示には良く反応しません
- Fortune 100 企業と小規模スタートアップの両方で働いた経験があります。どちらにも長所と短所があります
- 小さく段階的な成果物と、どんなタスクにも取り組む意欲を求めています
- 繰り返しタスクは自動化を好みます
- マネージャーだけが[リーダー](/handbook/leadership/)ではないと信じています

### 業務パターン

- [中部標準時](https://www.timeanddate.com/time/zones/ct)で生活・仕事をしています。
  - 冬は中部標準時（UTC-6）
  - 夏は中部夏時間（UTC-5）
- 朝は出発前に家族と過ごし、夕方も同様です。この時間帯は基本的に対応できません。事前に連絡をもらえれば対応できる場合もあります。
- 金曜日は [GitLab の Focus Fridays](/handbook/communication/#focus-fridays) に則り、ミーティングではなく集中した非同期作業のために確保しています
- 週末に多少作業することがあっても、他の人が通常業務時間外に働くことは一切期待しません
- 休暇では、できる限り（[インターネット接続がない場所](https://en.wikipedia.org/wiki/Burntside_Lake#/media/File:Burntside_Lake_BWCAW.jpg)に行くことも含めて）切断するよう努めます

### コミュニケーションの好み

- いつでも Slack やメールをしてもらって構いませんが、返信は業務時間内になる可能性が高いです。デバイスは夜間「通知なし」モードになっています。
- 業務時間外に Slack やメールを送ることがありますが、次の業務日まで返信は期待しません。忘れないうちに書き留めておく必要があるだけのことが多いです。
- 特定の時間内に返信が必要な場合は教えてください。例えば「ミーティング前にこれをレビューしてもらえますか？」「今日中に教えてもらえますか？」のように。
- GitLab では `@mnohr` のメンションのみ通知されます

### フィードバック

- 全員からのフィードバックを歓迎しており、学んで改善するための方法と捉えています
- プライベートで、フィードバックであることを明示してください。明確かつ具体的にしてもらえると効果的です
