## Issue 管理 & プランニング

### Issue の準備状況確認

[Issue ステータスリファレンス](index.md#issue-status-reference)に従って、Issue が開発に向けて準備できているかを評価するためにこのプロンプトを使用します:

```markdown
以下のチェックリストに基づいてこの Issue を評価し、推奨事項を提供してください

### 主要要件
- [ ] **コンテキスト**: チーム/セクションがラベルで識別されているか?
- [ ] **問題**: 現在の状態と望ましい状態が明確か?
- [ ] **アクション**: チェックボックスまたは明確な選択肢を伴う次のステップがあるか?
- [ ] **リンク**: 親 Epic/イニシアチブが参照されているか?
- [ ] **具体性**: 既知の場合は正確なコンポーネント/テーブル/インデックス/設定名が提供されているか?

### ディスカッション Issue が持つべき要素
- [ ] **質問**: 入力が必要な具体的な項目がリストアップされているか?
- [ ] **ステークホルダー**: 関与理由とともにタグ付けされているか?
- [ ] **タイムライン**: 決定またはアクションのタイムラインが示されているか?

### 存在すべきラベル
- [ ] カテゴリラベル（例: `Category:Cell`）
- [ ] チーム/グループラベル（例: `group::cells infrastructure`）
- [ ] タイプラベル（例: `type::maintenance`）

### コミュニケーション
- [ ] リソース/ドキュメントがリンクされているか?

## スコア
- **準備完了**: 80%以上チェック済み
- **要改善**: 50〜79%チェック済み
- **未準備**: 50%未満チェック済み
```

### Issue の更新

私たちの[開発ワークフロー](index.md#development-workflow)の一部として、週次の Issue 更新に使用するプロンプトです:

```markdown
Issue のディスカッションと MR の進捗を確認し、直近 7 日間の活動の非常に簡潔なサマリーをリンク付きで提供してください。
```

### クローズ時の Issue サマリー

```markdown
クローズ時に Issue の説明に記載する簡潔なサマリーを提供してください。研究、議論、またはデータが参照されたスレッドへの参照を含め、適切な場合はサマリーテーブルを作成してください。
```

## 開発 & コードレビュー

### 手動テスト計画の作成

このプロンプトはマージリクエストのテスト計画を作成するのに役立ちます。通常、テストアプローチを指定するための改善が必要です:

```markdown
このマージリクエストの手動テスト計画を作成して、自分で実行できるようにしてください。
```

特定のシナリオ向けの改善例:

```markdown
追加された新しい protofile に対して grpcurl を使った手動テスト計画を作成してください。
```

### コードレビューと説明

コードレビュー中に変更を理解するためにこのプロンプトを使用します:

```markdown
ファイル X の変更はなぜ必要ですか？何をしているのですか？
```

より包括的な分析のための代替プロンプト:

```markdown
これらの変更が何をしているのか、なぜ必要なのか、そして現在のアプローチに対してどのような他の選択肢があるかを説明してください。
```

### ドキュメントの更新

マージリクエスト中に、ドキュメントの更新が必要かどうかを確認します:

```markdown
どこかのドキュメントを更新する必要はありますか？
```

## チームレポーティング & コミュニケーション

### 週次 Epic サマリー

私たちの[チームミーティング](index.md#team-meetings)やグランドレビューの更新向けにステータスレポートを生成するためにこのプロンプトを使用します:

```markdown
このマークダウンテンプレートを使用して、Cells Infrastructure の最新の Epic 更新を基に今週のサマリーを作成してください。リンクを含めてください。

2025-xx-xx Update for Grand Review
:issue-blocked: Help Needed / Blockers

None this week

:closed_book: To Be Closed

None this week

:star2: Highlights

None this week

```
